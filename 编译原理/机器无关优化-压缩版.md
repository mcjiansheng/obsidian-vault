# 文件名：机器无关优化.md（压缩版）

## 1. 本章核心框架
机器无关优化在 IR 层进行，不依赖具体目标机器，基于程序语义和控制流/数据流进行等价变换。优化的基础是**数据流分析**。

```
优化后的IR ← 公共子表达式消除 + 复制传播 + 死代码删除 + 常量折叠/传播 + 代码移动 + 强度削减
              ↑ 都依赖数据流分析：到达定值 / 活跃变量 / 可用表达式
```

## 2. 必背概念

- 【定义】基本块 Basic Block：①只能从第一条进入 ②除最后一条外无跳转 ③中间顺序执行
- 【定义】入口语句 Leader：①程序第一条 ②跳转目标 ③跳转语句之后
- 【定义】流图/CFG Control Flow Graph：节点=基本块，边=控制流转移
- 【定义】公共子表达式 Common Subexpression：E 之前已计算且分量未变 → 可复用
- 【定义】复制传播 Copy Propagation：`u=v` 后可用 v 替换 u
- 【定义】死代码 Dead Code：赋值后变量不再被使用 → 可删除
- 【定义】常量折叠 Constant Folding：编译时直接计算常量表达式
- 【定义】常量传播 Constant Propagation：将已知常量值传播到使用处
- 【定义】代码移动 Code Motion：循环不变代码外提 LICM
- 【定义】归纳变量 Induction Variable：每次赋值增加固定量的变量
- 【定义】强度削减 Strength Reduction：昂贵运算 → 便宜运算（如乘法→加法）
- 【定义】支配关系 Dominate：d dom n = 去 n 必先经 d
- 【定义】回边 Back Edge：n→d 且 d dom n
- 【定义】自然循环 Natural Loop：由回边 n→d 确定的循环（d=循环头；n 和能到 n 且不经 d 的节点）
- 【定义】前置块 Preheader：循环外提代码放置处，在进入循环前执行一次

## 3. 数据流分析核心方程

### 3.1 通用形式
```
IN[B]  = 进入基本块B前成立的信息
OUT[B] = 离开基本块B后成立的信息
gen[B] = B内生成的信息
kill[B] = B内杀死的信息

OUT[B] = gen[B] ∪ (IN[B] - kill[B])
```

### 3.2 三大数据流分析对比

| 分析 | 方向 | May/Must | 汇合运算 | 方程 |
|------|------|----------|---------|------|
| 到达定值 | 前向 | May | ∪ | IN[B]=∪OUT[P]; OUT=gen∪(IN-kill) |
| 活跃变量 | 反向 | May | ∪ | OUT[B]=∪IN[S]; IN=use∪(OUT-def) |
| 可用表达式 | 前向 | Must | ∩ | IN[B]=∩OUT[P]; OUT=e_gen∪(IN-e_kill) |

### 3.3 到达定值 Reaching Definitions

【定义】定值 d 到达点 p：从 d 到 p 存在一条路径且变量未被重新定值

【公式】
```
IN[B]  = ∪ OUT[P]    P ∈ pred(B)
OUT[B] = gen[B] ∪ (IN[B] - kill[B])
```

【应用】判断变量使用可能来自哪些赋值

### 3.4 活跃变量 Live Variables

【定义】变量 x 在点 p 活跃：从 p 出发存在某路径使用 x 且未被重新定值

【公式】
```
OUT[B] = ∪ IN[S]     S ∈ succ(B)
IN[B]  = use[B] ∪ (OUT[B] - def[B])
```

【应用】死代码消除、寄存器分配（冲突图构造）

### 3.5 可用表达式 Available Expressions

【定义】E 在点 p 可用：从入口到 p 的所有路径都计算过 E 且分量未变

【公式】
```
IN[B]  = ∩ OUT[P]    P ∈ pred(B)    ← Must分析，用 ∩
OUT[B] = e_gen[B] ∪ (IN[B] - e_kill[B])
```

【应用】全局公共子表达式消除

### 3.6 数据流分析迭代算法

【算法】
```
初始化所有 IN/OUT
while (有集合发生变化):
    按某种顺序遍历基本块
    根据数据流方程更新 IN/OUT
```
直到**不动点 fixed point**（所有集合不再变化）

## 4. 核心优化速查

| 优化 | 依赖分析 | 基本原理 |
|------|---------|---------|
| 公共子表达式消除 | 可用表达式 | E 之前算过+分量未变 → 复用 |
| 复制传播 | 到达定值 | u=v → 后面用 v 替换 u |
| 死代码删除 | 活跃变量 | 赋值左部不活跃+无副作用 → 删除 |
| 常量折叠 | — | 编译时算常量表达式 |
| 常量传播 | 到达定值 | 已知常量值替换变量 |
| 循环不变代码外提 | 循环识别+定值分析 | 循环中不变的表达式移出循环 |
| 强度削减 | 归纳变量识别 | 乘法 → 加法 |
| 归纳变量消除 | 归纳变量识别 | 消除多余归纳变量 |

## 5. 典型题型与解题模板

### 题型1：划分基本块 + 构造流图
- 识别特征：给一段三地址码，要求划分基本块并画流图
- 解题步骤：①找 leader（第一条/跳转目标/跳转后）②每个 leader 到下一个 leader-1 为一块 ③按控制流画边
- 常见陷阱：return/halt 后无后继块

### 题型2：到达定值分析
- 识别特征：给 CFG，求每个基本块的 gen/kill/IN/OUT
- 解题步骤：①找每个块内的定值(gen)和被杀死的定值(kill) ②初始化 IN=∅ ③迭代 IN[B]=∪OUT[P], OUT=gen∪(IN-kill) ④直到不动点
- 常见陷阱：May 分析用 ∪（并集）

### 题型3：活跃变量分析
- 识别特征：给 CFG，求每个块的 use/def/IN/OUT
- 解题步骤：①找 use（先使用后定义的变量）和 def（被定义的变量）②反向分析 ③初始化 IN=∅ ④迭代 OUT=∪IN[S], IN=use∪(OUT-def)
- 常见陷阱：方向是反向（从出口往回推）；May 分析用 ∪

### 题型4：可用表达式分析
- 识别特征：给 CFG，求可用表达式集合
- 解题步骤：①找 e_gen（计算且在块尾仍可用的表达式）和 e_kill（分量被重定义）②前向分析 ③⚠️ Must 分析：IN=∩OUT[P]（交运算！）

### 题型5：识别循环 + 代码外提
- 解题步骤：①求支配关系 ②找回边 n→d(d dom n) ③确定循环体 ④判断循环不变表达式 ⑤外提到 preheader

## 6. 易错点与速记

- 【易错】到达定值是 May 分析（∪），可用表达式是 Must 分析（∩）— 这是最容易混淆的！
- 【易错】活跃变量是反向分析（从出口往回），其余两个是前向分析
- 【易错】死代码删除不能删有副作用的语句（如函数调用、可能抛异常的操作）
- 【易错】可用表达式用 ∩（Must）：必须所有路径都计算过才算可用
- 【易错】常量传播要考虑控制流：分支路径上变量可能不是同一常量
- 【考点】数据流分析必须迭代到不动点，因为有循环
- 【速记】支配关系：d dom n = 去 n 必先经 d；回边：n→d 且 d dom n

## 7. 打印版极简总结
```
优化基础 = 基本块 + CFG + 数据流分析(迭代至不动点)
三大分析：
  到达定值(前向/May/∪) → 复制传播、常量传播
  活跃变量(反向/May/∪) → 死代码删除、寄存器分配
  可用表达式(前向/Must/∩) → 公共子表达式消除
六大优化：公共子表达式消除、复制传播、死代码删除、常量折叠/传播、LICM、强度削减
循环识别：支配关系 + 回边 → 自然循环 → preheader外提
```
