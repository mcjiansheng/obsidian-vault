# 文件名：语义分析.md（压缩版）

## 1. 本章核心框架
语义分析在 AST 上计算属性、检查类型、维护符号表。核心工具是语法制导定义(SDD)和语法制导翻译(SDT)。

```
语法树 → 属性计算(SDD/SDT) → 类型检查 + 符号表维护 → 带类型信息的AST
         ↑ 综合属性(自底向上)     ↑ S属性SDD → 适合LR分析
         ↑ 继承属性(自顶向下)     ↑ L属性SDD → 适合LL分析
```

## 2. 必背概念

- 【定义】SDD Syntax-Directed Definition：CFG + 属性 + 语义规则
- 【定义】SDT Syntax-Directed Translation：产生式体中嵌入语义动作的 CFG
- 【定义】综合属性 Synthesized Attribute：从子节点向父节点传递（如 E.val, E.type）
- 【定义】继承属性 Inherited Attribute：从父节点或左兄弟向当前节点传递（如声明类型信息）
- 【定义】S 属性 SDD：只含综合属性，天然适合自底向上/LR 分析
- 【定义】L 属性 SDD：允许综合属性 + 受限继承属性（只能从父节点或左兄弟继承）
- 【定义】依赖图 Dependency Graph：属性实例之间的依赖关系，需按拓扑序计算
- 【定义】类型表达式 Type Expression：描述类型结构（基本类型、array、function、pointer、record）
- 【定义】结构等价 Structural Equivalence：比较类型表达式树结构是否相同
- 【定义】命名等价 Name Equivalence：比较是否来自同一声明
- 【定义】类型兼容 Type Compatibility：允许自动类型转换（coercion）
- 【定义】Widening：安全拓宽（如 int→float）
- 【定义】Narrowing：可能丢失信息的窄化（如 float→int）

## 3. 关键算法 / 规则 / 公式

### 3.1 S 属性 SDD → 后缀 SDT

【算法】综合属性动作放在产生式最右端：
```
E → E₁ + T  { E.val = E₁.val + T.val }
```
在 LR 分析中，归约时计算：右部符号属性已在栈中，直接计算左部属性。

### 3.2 L 属性 SDD → SDT

【算法】
- 计算继承属性的动作：放在对应符号出现之前
- 计算综合属性的动作：放在产生式最右端
```
A → X { Y.inh = f(X.syn) } Y { A.syn = g(Y.syn) }
```

### 3.3 消除左递归时语义规则保持等价

【算法】引入继承属性 inh（累积结果）和综合属性 syn（返回结果）：
```
原：E → E₁ + T  {E.val = E₁.val + T.val}
   E → T        {E.val = T.val}

消左递归后：
E  → T E'              { E'.inh = T.val; E.val = E'.syn }
E' → + T E₁'           { E₁'.inh = E'.inh + T.val; E'.syn = E₁'.syn }
E' → ε                 { E'.syn = E'.inh }
```

### 3.4 声明处理：width 与 offset

【算法】局部变量连续分配：
```
offset = 0
对每个声明：
    top.put(name, type, offset)
    offset += width(type)
```

### 3.5 LR 栈中属性计算

【算法】
1. 栈中每个状态附带属性值
2. 归约 `A → X Y Z` 时：Z 属性在栈顶，Y 在 top-1，X 在 top-2
3. 计算 A 的属性后压回栈

### 3.6 类型综合规则

```
E₁: int, E₂: float ⇒ E₁ + E₂ : float (max 类型)
```
隐式转换：`t1 = (float)i; t2 = x fmul t1`

## 4. 典型题型与解题模板

### 题型1：判断 SDD 是 S 属性还是 L 属性
- 识别特征：给出产生式和语义规则
- 解题步骤：①看是否只有综合属性(S属性) ②若有继承属性，检查是否只依赖父节点或左兄弟(L属性) ③画依赖图检查是否有环
- 常见陷阱：继承属性依赖右兄弟 → 不是 L 属性

### 题型2：消除左递归并改写语义规则
- 识别特征：给出含左递归的文法和语义规则
- 解题步骤：①消左递归 ②引入继承属性 inh 保存累积结果 ③引入综合属性 syn 返回结果 ④重写语义规则
- 常见陷阱：忘记处理 ε 产生式的属性传递

### 题型3：声明处理 — 计算类型宽度和偏移
- 识别特征：给出类型声明序列，要求计算布局
- 解题步骤：①确认各类型宽度 ②按声明顺序累加 offset ③记录类型 → 符号表
- 常见陷阱：对齐和补白（题目若不要求可不考虑）

### 题型4：类型表达式推导
- 识别特征：给表达式和类型，要求推导结果类型
- 解题步骤：①自底向上综合类型 ②遇到类型不匹配时应用 widening/narrowing
- 示例：`int[10] → array(10, int)`；`float f(int,int) → int×int→float`

### 题型5：逆波兰式(RPN)翻译
- 识别特征：中缀表达式转后缀
- 解题步骤：操作数直接输出，运算符放到操作数之后
- 示例：`a+b*c → a b c * +`

## 5. 易错点与速记

- 【易错】SDD 中若有循环依赖（如 A.s = B.i, B.i = A.s + 1），无法计算
- 【易错】S 属性 SDD 只能自底向上/LR；L 属性 SDD 可自顶向下/LL
- 【易错】继承属性不能依赖右侧兄弟，否则不是 L 属性
- 【易错】语义动作的副作用（修改符号表、生成代码）必须在依赖信息就绪后执行
- 【速记】S属性=纯综合，自底向上；L属性=综合+受限继承，从左到右
- 【速记】类型转换：隐式(automatic) vs 显式(cast)；widening(安全) vs narrowing(可能丢信息)

## 6. 打印版极简总结
```
语义分析 = SDD(属性+规则) → 依赖图(拓扑序) → 属性计算
S属性SDD：纯综合 → LR归约时算
L属性SDD：综合+受限继承 → LL/递归下降时算
核心：类型检查 + 符号表 + offset/width + 类型转换(widen/narrow)
```
