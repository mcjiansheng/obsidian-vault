### LL语法分析
![[Pasted image 20260620102610.png]]
**消除左递归**：
如果有
`A → Aα1 | Aα2 | ... | Aαm | β1 | β2 | ... | βn`
其中 `β` 不以 `A` 开头，那么改写为：
```
A  → β1 A' | β2 A' | ... | βn A'
A' → α1 A' | α2 A' | ... | αm A' | ε
```

例子：
`E -> E + T | T`
改为
```
E -> T E'
E' -> + T E' | ε
```
### 间接左递归
不一定是直接写成 `A -> Aα` 才叫左递归
例如：
```
A → B a
B → A b | c
```

在这有
`A ⇒ B a ⇒ A b a`
这就是间接左递归

消除间接左递归通常要
```
1. 给终结符排序 A1,A2,...,An
2. 按顺序替换前面非终结符的产生式
3. 把间接左递归转化成直接左递归
4. 再用直接左递归消除方法处理
```

**左公因子 Left Factoring**
如果两个产生式有相同前缀
`A -> αβ1 | αβ2`
自顶向下分析看到 α 时无法确定使用哪一条规则
因此需要提取左公因子：
```
A  → α A'
A' → β1 | β2
```

#### First 集合
**终结符**
`FIRST(a) = {a}`

**空串**
`FIRST(ε) = { ε }`

**非终结符**
如果
`A -> α`
则将
`FIRST(α) - {ε}` 加入 `FIRST(A)`
如果 `α` 能推出 `ε`，则把 `ε` 加入 `FIRST(A)`。

**符号串**
对于
`X1 X2 ... Xk`
先看`X1`
`把 FIRST(X1) - {ε} 加入`
如果 X1 能推出 ε ，再看 X2

#### FOLLOW集
**开始符号**
`$ ∈ FOLLOW(S)`
因为开始符号推导出的完整句子后面就是结束输入标记

**A → α B β**
那么
`FIRST(β) - {ε} 加入 FOLLOW(B)`
因为 `B` 后面接着 `β`，所以 `β` 的开头可能跟在 `B` 后面。

**β 能推出 ε**
如果 `A → α B β` 且 `β ⇒* ε`
那么 `FOLLOW(A) 加入 FOLLOW(B)`
因为 `β` 可以消失，所以 `B` 后面可能直接跟着原来 `A` 后面的符号。

#### 构造SELECT集合
公式：
```
SELECT(A → α) = FIRST(α) ∪ FOLLOW(A) 如果 ∈ FIRST(α)  
SELECT(A → α) = FIRST(α) 如果 ε ∉ FIRST(α)
```

也就是说：
如果α不能推出空串，看 FIRST(α)
如果α能推出空串，还要看 FOLLOW(A)
