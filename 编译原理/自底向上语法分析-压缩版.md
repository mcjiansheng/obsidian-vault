# 文件名：自底向上语法分析.md（压缩版）

## 1. 本章核心框架
自底向上分析从输入串出发，逐步归约为开始符号。核心框架是移进-归约分析，LR 是最重要的自底向上技术。LR(0)→SLR→LR(1) 精确度递增。

```
输入串 → 移进-归约(栈+分析表) → 归约至开始符号 → 接受
          ↑ 核心数据结构：LR(0)项集族 + ACTION/GOTO表
```

## 2. 必背概念

- 【定义】归约 Reduce：推导的反向操作，β 替换为 A（有 A → β）
- 【定义】句柄 Handle：当前右句型中应被归约的子串；出现在最右推导路径中
- 【定义】移进 Shift：把当前输入符号压入栈
- 【定义】LR(k)：从左到右扫描(L)，反向构造最右推导(R)，向前看 k 个符号
- 【定义】LR(0)项：产生式右部某位置加点，`A → α·β`（α已识别，β待识别）
- 【定义】增广文法：加入 `S' → S`，当归约到 `S' → S·` 时接受
- 【定义】CLOSURE(I)：若 `A → α·Bβ` 在闭包中，对 B 的所有产生式加入 `B → ·γ`，反复至收敛
- 【定义】GOTO(I, X)：I 中所有 `A → α·Xβ` 点右移后求闭包
- 【定义】SLR Simple LR：用 FOLLOW 集限制 LR(0) 的归约
- 【定义】LR(1)项：`[A → α·β, a]`，a 是向前看符号，归约仅当下一个输入为 a 时允许

## 3. 关键算法 / 规则

### 3.1 移进-归约分析框架

【算法】四个动作：
1. **移进 Shift**：输入符号压栈
2. **归约 Reduce**：栈顶句柄替换为非终结符（按产生式 A → β）
3. **接受 Accept**：栈中为 $S，输入为 $
4. **报错 Error**：无法移进/归约/接受

### 3.2 规范 LR(0) 项集族构造

【算法】
1. 增广文法：添加 `S' → S`
2. `I₀ = CLOSURE({S' → ·S})`
3. 对每个项目集 I 和每个文法符号 X，计算 `GOTO(I, X)`
4. 新项目集加入集合，重复至不再产生新集

### 3.3 SLR 分析表构造

【算法】对每个状态 Ii：
- 若有 `A → α·aβ` 且 `GOTO(Ii, a) = Ij`：`ACTION[i, a] = shift j`
- 若有 `A → α·`(A ≠ S')：对所有 `a ∈ FOLLOW(A)`，填 `ACTION[i, a] = reduce A → α`
- 若有 `S' → S·`：`ACTION[i, $] = accept`
- 对非终结符 A：`GOTO(Ii, A) = Ij` → `GOTO[i, A] = j`

### 3.4 LR 分析器运行

【算法】
1. 初始：栈 = `$ s₀`，输入 = `w$`
2. 循环查 `ACTION[栈顶状态, 当前输入符号]`：
   - `shift j`：移进符号，压入状态 j
   - `reduce A → β`：弹出 |β| 个状态，查 `GOTO[新栈顶, A]`，压入新状态
   - `accept`：成功
   - `error`：报错

## 4. 典型题型与解题模板

### 题型1：求句柄
- 识别特征：给推导步骤或句型，问"下一步该归约哪个子串？"
- 解题步骤：①判断当前句型是否为规范最右推导的某一步 ②找产生式右部匹配且位置正确的子串
- 【易错】句柄不是任意匹配产生式右部的子串，必须在规范推导路径中

### 题型2：构造 LR(0) 项集族
- 识别特征：给出文法，求所有 LR(0) 项目集（状态）
- 解题步骤：①增广文法 ②从 I₀=CLOSURE({S'→·S}) 开始 ③对每个 I 和每个 X 求 GOTO(I,X) ④重复至收敛
- 常见陷阱：CLOSURE 不彻底；GOTO 后忘记求闭包

### 题型3：构造 SLR 分析表
- 识别特征：给出文法，构造 SLR ACTION/GOTO 表
- 解题步骤：①构造 LR(0) 项集族 ②求 FOLLOW 集 ③按 SLR 规则填表
- 常见陷阱：归约项要查 FOLLOW(A)，不是所有终结符都填归约

### 题型4：判断是否 SLR 文法
- 识别特征：检查分析表是否有冲突
- 解题步骤：①构造 SLR 表 ②检查每个表项是否只有一条动作
- 常见陷阱：移进-归约冲突（同一表项既有 shift 又有 reduce）；归约-归约冲突

## 5. 易错点与速记

- 【易错】句柄不是任意产生式右部匹配，必须在最右推导路径中
- 【易错】LR(0) 做归约不看前瞻符号，太粗暴，容易冲突
- 【易错】SLR 用 FOLLOW 限制归约，但 FOLLOW 是全局集合，不够精确
- 【易错】归约-归约冲突比移进-归约冲突更严重
- 【考点】LR(1) 可识别所有确定性上下文无关文法，能分析的文法集合是 LL(k) 的超集
- 【速记】LR = 从左到右 + 反向最右推导 + 移进归约
- 【速记】LR(0)项的点：左边已识别，右边待识别

## 6. 打印版极简总结
```
自底向上 = 移进-归约框架：栈 + ACTION/GOTO表
LR(0)：项集族 = CLOSURE + GOTO，归约不看前瞻 → 常冲突
SLR：用 FOLLOW(A) 限制归约 → 减少冲突
LR(1)：项带向前看符号 [A→α·β, a] → 最精确
核心：句柄识别 + 归约 = 最右推导的逆
```
