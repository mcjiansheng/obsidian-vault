[[语法分析]]

**什么是自底向上**
自底向上分析从输入串开始，逐步把子串归约成非终结符，最后归约成开始符号

在构造语法树时
```
从叶子到根
从小结构到大结构
```


重要的通用框架是
移入-归约
LR是最大的考验构造出移入-归约语法分析器的语法类

**归约是推导的反向操作**
如果有产生式：
A → β

自顶向下推导是：

```
A ⇒ β
```

自底向上归约则反过来：

```
β ⇒ A
```
一次归约实质上就是一次推导的反向操作；整条自底向上归约序列等价于某个最右推导的逆序。

## 句柄Handle

**什么是句柄**
在某个右句型中，如果某个子串`β`是最近一次最右推导产生出来的部分，并且有产生式：

```
A → β
```

那么这个 `β` 就是当前应该归约的句柄。

> 句柄就是当前这一步应该被折叠成非终结符的那一段子串

句柄是一个二元组`(A → β , 位置)`：把该位置上的β替换为A后，恰好得到最右推导中的上一步句型

**句柄不是“任意能匹配产生式右部的子串”**

例如：

```
T * id
```

里面：

```
T 可以匹配 E → T
id 可以匹配 F → id
```

那能不能先把 `T` 归约成 `E`？

不能。

因为：

```
E * id
```

不是合法的右句型。

所以当前真正的句柄应该是右边的 `id`，先归约为 `F`。

> 句柄并不是任意右部匹配，而是必须出现在规范最右推导路径中的子串；自底向上分析不是乱归约，而是在逆行一条最右推导

## 移进 - 归约分析框架
移进-归约分析使用
```
一个栈
一个输入缓冲区
一张分析表或一套决策规则
```

初始状态：
```
栈：$
输入：w$
```

成功状态：
```
栈:$S
输入:$
```

主要动作有四种

**移进 Shift**
把当前输入符号移入栈顶
```
栈：$ id
输入：* id $
```

**归约 Reduce**
如果栈顶某段符号是句柄，就用产生式左部替换它
例如：
F -> id
则：
栈：$ id
归约为:
栈：$ F

**接受 Accept**
当栈中已经归约成开始符号，并且输入结束，则分析成功

**报错 Error**
如果当前状态无法移进、无法归约、无法接受、则语法错误

## 移进-归约冲突与归约-归约冲突

**移进-归约冲突 Shift-Reduce Conflict**
分析器不知道应该
```
继续移进？
还是现在归约？
```

例子 dangling else:
```
S → if E then S
  | if E then S else S
  | other
```

输入
`if E then if E then other else other`

当栈中为`if E then if E then other`
而输入符号为`else`

此时有两种可能
```
归约：把 if E then other 归约为S
移进：把 else 移进来，形成 if E then S else S
```

**归约-归约冲突 Reduce-Reduce Conflict**
分析器不知道应该用哪条产生式归约
例如某个栈顶符号串同时可以按两条规则归约
```
A → α
B → α
```

此时就会产生归约-归约冲突

归约-归约冲突通常比移进-归约冲突更为严重，因为它说明文法结构本身更加不准确

## LR语法分析

LR是最重要的自底向上分析技术

1. LR(k) 的含义
```
L：Left-to-right，从左到右扫描输入
R：Rightmost derivation in reverse，反向构造最右推导
k：最多向前看 k 个符号
```

> 从左到右扫描输入，通过归约反向构造最右推导，并最多看k个前瞻符号

k增大时分析器规模会急剧增大，因此实际重点关注 k=0 和 k=1

**LR分析的优点**
```
能处理的文法范围大
无回溯
可以较早发现语法错误
适合自动生成分析器
```

LR 是非常通用的无回溯移入-归约技术，LR(1) 可以识别所有确定性上下文无关文法，而且能分析的文法集合是 LL(k) 文法的超集。

## LR(0)项

**什么是LR(0)项**
LR(0)项就是中产生式右部的某个位置加一个点
`A → α · β`

点表示
```
α 已经识别了；
β 还没有识别。
```

例如：
`E -> E · + T`
代表：
```
已经识别出 E；
接下来期待看到 + T。
```

`F → id ·`
点在最右边，表示：
```
id 已经完整识别，可以把 id 归约为 F。
```

A  → ε 只对应一个项 A →.

**项集 Item Set**
一个状态通常不是一个项目，而是一组项目
例如
```
I0:
E' → · E
E  → · E + T
E  → · T
T  → · T * F
T  → · F
F  → · (E)
F  → · id
```
表示：
```
现在可能要识别 E；
而 E 可能从 E+T 开始，也可能从 T 开始；
T 又可能从 T*F 或 F 开始；
F 又可能从 (E) 或 id 开始。
```

### 增广文法
为了让分析器知道什么时候接受输入，需要引入新的开始符号：
原文法开始符号是：
`S`
增广文法加入：
`S' → S`
当分析器最后归约到：
`S' → S ·`
并且输入结束 `$`，就可以接受。
代表分析结束

### CLOSURE 和 GOTO

**CLOSURE闭包**
如果项集中有
`A → α · B β`
点后面是非终结符 `B` ，说明接下来要识别一个B
那么B可能会由其他产生式展开
```
B → γ1
B → γ2
```
因此需要把
```
B → · γ1
B → · γ2
```
加入相集，不断重复，直到不能再加入新项目
这就是 `CLOSURE(I)`

> 如果 `A → α · B β` 在闭包中，则对 `B` 的任意产生式 `B → γ`，加入 `B → · γ`，直到收敛。

**GOTO 转换**
`GOTO(I,X) 表示`
> 在状态 I 中，如果接下来识别了文法符号 X，会进入哪个新状态？

如果`I`中有
`A → α · X β`
读入 `X` 后点右移：
`A → α X · β`

对所有这种项目做点右移再求闭包，就能得到：
`GOTO(I, X)`

![[Pasted image 20260610211047.png]]

**规范 LR(0) 项集族**
```
1. 构造增广文法 G'
2. I0 = CLOSURE({S' → · S})
3. 对每个项目集 I 和每个文法符号 X，计算 GOTO(I, X)
4. 如果得到新项目集，就加入集合
5. 重复直到不再产生新项目集
```
得到的所有项目集就是规范 LR(0) 项集族，也就是 LR(0) 自动机的状态集合。

## LR分析表 ACTION / GOTO
```
ACTION 表：面对终结符时做什么
GOTO 表：归约成非终结符后跳到哪里
```
**1. ACTION 表**
可能动作：

```
shift j     移进并进入状态 j
reduce A→α  按产生式 A→α 归约
accept      接受
error       报错
```

**2. GOTO表**
如果刚刚归约出非终结符 A，并且当前栈顶状态是 i，则差：
`GOTO[i,A]`
得到新状态并压栈

**3.LR分析器工作过程**
栈中主要存状态
如果：
`ACTION[i,a] = shift j`
则
```
移进 a
压入状态 j
```

如果
`ACTION[i, a] = reduce A → α`
则
```
弹出 |α| 个状态
设当前栈顶状态为 t
查 GOTO[t, A] = j
压入 j
```

如果
`ACTION[i, $] = accept`
则分析成功

## LR(0)的局限性
LR(0)最大的问题：
> 它做归约决策时不看任何前瞻符号

也就是说，一旦某个状态中有完成项：
```
A → α ·
```
LR(0) 就倾向于在所有输入符号下都归约。
这太粗暴了。
例如状态中同时有：
```
T → F ·
T → T · * F
```
如果下一个符号是 `*`，应该继续移进，而不是把 `T` 归约掉。
> LR(0) 只看项目中的点位置，不知道后面真正可能出现的符号，因此在“能归约”和“能移进”同时存在时，无法做唯一选择。

## SLR分析
Simple LR
在 LR(0) 的基础上加入 FOLLOW 集来限制归约。

**核心思想**
如果状态中有完成项：
```
A → α ·
```
LR(0) 会在所有终结符下归约。

SLR 改成：
```
只有当前输入符号 a ∈ FOLLOW(A) 时，才按 A → α 归约。
```
这样可以减少冲突。

### SLR表构造
对规范状态LR(0)项集族中的每个状态 `Ii` ：
如果有：
`A → α · a β`
其中a是终结符，且：
`GOTO(Ii, a) = Ij`
则`ACTION[i, a] = shift j`

如果有完成项：
`A → α ·`
且 `A ≠ S'`，则对所有：
`a ∈ FOLLOW(A)`
填：
`ACTION[i, a] = reduce A → α

如果有：
`S' → S ·`
则：
`ACTION[i, $] = accept`

对于非终结符 `A`：
`GOTO(Ii, A) = Ij`
则：
`GOTO[i, A] = j`

### SLR的优缺点

优点：
```
比 LR(0) 强
状态数仍然和 LR(0) 一样
构造简单
```

缺点：
```
FOLLOW(A) 是全局集合，不够精确
有时仍然会产生冲突
```

## LR(1) 分析
LR(1) 比 SLR 更精确。(只需理解)

**LR(1)项**
记作
`[A → α · β, a]`
其中 `a` 是向前看符号 lookahead
代表：如果将来要归约 `A → α`，只有当下一个输入符号是 `a` 时，才允许归约。
