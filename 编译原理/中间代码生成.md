**中间代码生成到底在做什么**？
编译器前端先做词法分析、语法分析、语义分析，得到语法树、符号表、类型信息等；
中间代码生成器再把这些结构翻译成一种接近机器但还不绑定具体机器的程序表示，也就是IR

**IR Intermedia Representation**
编译器在前端与后端之间使用的、与具体源语言和目标机器都相对独立的程序表示形式，用于承载程序语义、驱动优化，并作为目标代码生成的输入

```
源代码
  ↓ 词法分析
Token 序列
  ↓ 语法分析
语法树 / AST
  ↓ 语义分析
带类型、符号绑定的 AST
  ↓ 中间代码生成
IR / 三地址码 / SSA / 字节码等
  ↓ 优化
优化后的 IR
  ↓ 目标代码生成
汇编 / 机器码
```
所以，中间代码生成不是“最终代码生成”，它生成的是给**优化器和后端使用的中间程序**
例如：
```C
x = a + b * c;
```
前端看到的是表达式树：
```
      =
    /   \
   x     +
        / \
       a   *
          / \
         b   c
```
中间代码生成后可能变成：
```
t1 = b * c
t2 = a + t1
x  = t2
```
这个形式比AST更线性、更容易分析，也比汇编更抽象、更容易移植

**为什么需要IR**？

核心原因：
**解耦与可移植性**、**优化平台**、**简化实现**、**跨语言统一**

IR让前端和后端解耦
同一前端可面向多个后端；同一后端可复用多个前端

IR是主要的优化平台
常量传播、死代码删除、公共子表达式消除、循环优化、赋值传播等通常不直接在源代码或汇编上做，而是在IR做

IR简化实现
AST太接近源语言，汇编太接近机器；IR处于中间层，既保留足够语义，又便于规则化分析

IR支持跨语言统一
LLVM就是典型例子：C/C++ 、Rust等语言可以先降到LLVM IR，再复用LLVM的优化器和后端

## IR的层次
**高层IR**
接近源语言，还保留函数、循环、异常、对象、闭包等高级结构
适合做高层优化，例如函数内联、异常消除、对象去虚拟化
离机器比较远，不适合直接做寄存器分配和指令选择

**中层IR**
最常见的课程重点， 例如三地址码TAC、四元式、SAA形式的TAC
既足够接近机器，方便优化和代码生成，又保留必要的结构信息

**底层IR**
更接近目标机，显式出现寄存器、栈、load/store、跳转、调用约定等
容易映射到机器指令，但抽象层级低，优化空间相对受限

### 设计IR考虑的维度

**控制流表示**
可以保留结构化控制流，例如 while、for、if-else；
有可能降成显式标签和goto，形成基本块与控制流图CFG

**数据表示**
可以采用三地址临时变量模型，也可以采用栈操作数模型
比如三地址码写成：
```
t1 = a + b
t2 = t1 * c
```
而栈式IR可能类似：
```
load a
load b
add
load c
mul
```

**赋值模型**
可以说普通赋值，也可以说SSA，静态单赋值
普通赋值中一个变量可以被多次赋值
SSA中每个版本只赋值一次

**内存与地址**
IR可以显式区分 load/store ，也可以更隐式；
也可以区分左指 l-value 和右值 r-value，这对数组、指针、结构体字段访问尤其重要

**类型与语义信息**
IR可以强类型，也可以弱类型；
可以保留异常、越界检查、GC屏障、并发原语等高级语义，也可以在较低层丢掉这些结构

**可分析性与可变换性**
好的IR要方便构建CFG、DFG，方便做数据流分析、短路求值和回填

### 常见IR范式

代表性IR：树形IR、三地址码、SSA、堆栈式IR、LLVM IR / Sea-of-Nodes 等

**AST/语法树注释** 属于树形IR，适合前端和早期高层变换，但不适合底层优化。
比如 a + b \* c 的树结构天然表示优先级，但不方便直接做线形数据流分析

**三地址码 TAC** 是本章核心，典型格式是：
```
x = y op z
if x goto L
param x
call f, n
return x
```

**SSA**是在 TAC 基础上加入版本号和 φ 函数的形式，非常适合优化。

**堆栈式 IR**把操作数隐含在栈上，例如 JVM 字节码。它简单、紧凑，但数据依赖不如三地址码显式。

**LLVM IR、GIMPLE、Sea-of-Nodes**等是工业编译器中广泛使用的 IR。课件提到 LLVM、GCC、Java HotSpot、Rustc 都使用 SSA 或其变体。

### 三地址码 TAC
三地址码的核心思想：**把复杂表达式拆成一条条简单指令，每条指令右边最多只有一个运算符**
三地址码通常从 AST 生成，是前端和后端之间的桥梁；它类似低级汇编，但独立于具体机器架构

```
result = arg1 op arg2
```
其中地址可以是
```
源程序名字：a, b, sum
常量：0, 1, 10
临时变量：t1, t2, t3
标签：L1, L2
```
例如：
```
x = (a + b) * (c - d) + e;
```
可以生成：
```
t1 = a + b
t2 = c - d
t3 = t1 * t2
t4 = t3 + e
x  = t4
```

三地址码的优点是：
```
简单：每条指令只做一个操作
线性：方便遍历和存储
控制流清晰：用 label 和 goto 表示分支循环
平台无关：不依赖 CPU 寄存器
利于优化：常量传播、死代码删除、公共子表达式消除都容易做
```
缺点也很明显：它会产生很多临时变量和冗余指令，但这些荣誉恰好可以交个后续优化器处理

### 三地址码的主要指令类型

**算数/逻辑运算**
```
x = y op z
t1 = a + b
t2 = t1 * c
```

**赋值与拷贝**
```
x = y
x = -y
sum = 0
```

**控制流**
```
goto L
if x relop y goto L
```

**函数调用**
```
param x
param y
call f,2
t = call g,1
```

**数组和指针访问**
```
t1 = i * width
t2 = a + t1
x = a[i]
a[i] = y
```
### 四元式、三元式、间接三元式
三地址码是一种逻辑形式，具体实现时可以用不同数据结构表示

**四元式**
四元式有四个字段
```
(op, arg1, arg2, result)
例如 x = y + z
可表示为 (+, y, z, x)
```
四元式的好处是结果变量显式存在，移动、删除、插入指令时比较方便。

**三元式**
三元式只有
```
(op, arg1, arg2)
它不用 result 字段，而是用“该三元式的位置”表示结果。
t1 = a + b
t2 = c - d
t3 = t1 * t2

三元式可以写为
(1) + a b
(2) - c d
(3) * (1) (2)
其中（1）表示第1条三元式的结果
```
三元式有一个严重的问题：
如果优化时移动或删除三元式，位置编号会变化，引用关系也须同步修改

**间接三元式**
间接三元式用一个“指针表”指向真正的三元式
优化时移动的时指针表，而不是三元式本身

这样既保留三元式简洁的优点，又避免移动三元式导致编号失效

### SSA 静态单赋值形式
SSA，Static Single Assignment
要求每个变量版本只赋值一次

SSA常用于优化，因为它让每个使用点对应唯一的定义点，方便做死代码删除、常量传播、寄存器分配等

普通代码：
```c
if (flag)
	x = -1;
else
	x = 1;
	
y = x * a;
```
SSA形式
```
if (flag)
    x1 = -1
else
    x2 = 1

x3 = φ(x1, x2)
y1 = x3 * a
```
  这里 `φ` 函数表示：程序从哪个控制流路径过来，就选哪个值。它不是 CPU 指令，而是 IR 中的抽象伪指令；
  最终生成机器码时需要消除 φ，通常在前驱基本块末尾插入 move 指令。

循环中也需要 φ 
```c
result = 1;
while (n > 1) {
    result = result * n;
    n = n - 1;
}
return result;
```
SSA 中循环头需要合并“初始值”和“上一次循环体更新后的值”
```
Header:
result1 = φ(result0, result2)
n1      = φ(n0, n2)
t1 = n1 > 1
if t1 goto Body else Exit

Body:
result2 = result1 * n1
n2 = n1 - 1
goto Header
```

### 类型与声明：为什么中间代码生成必须关心类型

课件进入“类型和声明”后，先讲类型检查。类型信息的用途包括：查错、确定名字需要的内存空间、计算数组元素地址、类型转换、选择正确运算符。

例如
```c
x * i
```
如果x是float，i是int，那么不能简单生成整数乘法，需要：
```
t1 = (float) i
t2 = x fmul t1
```
类型信息决定：
```
用整数加法还是浮点加法
变量占多少字节
数组元素宽度是多少
是否要插入类型转换
函数调用参数是否合法
```

**类型表达式**
类型表达式用来描述类型结构
常见的类型构造：
```
基本类型：int, float, char, bool
数组：array(n, T)
函数：T1 → T2
指针：pointer(T)
记录 / 结构体：record{f1:T1, f2:T2, ...}
别名 / typedef
```

**类型等价与类型兼容**

类型等价的两种常见定义
```
结构等价：两个类型表达式的树结构相同
命名等价：两个类型来自同一个声明或别名
```

类型兼容关注“能不能一起用”，例如
```c
int + float
```
虽然 int 和 float 不是同一类型，但 C 语言允许自动算术转换，所以表达式结果为 float

**静态检查、动态检查、类型推导**
静态检查在编译期完成，典型如C、Java；
动态检查在运行期完成，典型如Python、JavaScript。

类型推导：利用类型规则在语法树上自底向上推导
例如：
```
E1: int, E2:float
=> E1 + E2 : float
```

函数调用：
```
若 f : T1 -> T2
且参数类型为 T1
则 f(x) 但类型为 T2
```
### 声明处理：局部变量布局、类型宽度、offset
中间代码生成不仅要知道变量叫什么，还要知道变量存在在哪里
局部变量名的存储布局：变量的类型决定它需要的内存大小，即类型宽度
函数局部变量通常分配在一段连续区域中，因此每个变量可以获得一个相对地址的offset；变量类型和offset都保存在符号表中

```
int a;
float b;
int c;
假设int 宽度4 ,float 宽度8,则可以分配：
a: type=int,   offset=0
b: type=float, offset=4
c: type=int,   offset=12
```
对于记录类型 record，需要为字段创建单独符号表。处理 `record { D }` 时，先创建新符号表并压栈，然后处理字段声明 D，把字段加入新表，最后根据这个符号表构造 record 类型表达式，再出栈。

### 表达式翻译：code 、addr 、newTemp 、gen
表达式翻译的目标是把表达式编程三地址指令序列
表达式设置有两个重要属性：code 表示生成的代码，addr 表示表达式结果存放的地址，通常是临时变量；
newTemp() 生成临时变量，gen(...) 生成指令

例如产生式
```
E -> E1 + E2
```
语义规则为
```
E.addr = newTemp()
E.code = E1.code || E2.code ||
         gen(E.addr '=' E1.addr '+' E2.addr)
```

先生成做表达式代码，再生成右表达式代码，最后生成加法指令
```c
x = a + b * c
```
翻译过程：
```
E1 = a
E2 = b * c
```
生成
```
t1 = b * c
t2 = a + t1
x = t2
```
课件还给了赋值语句
```
S -> id = E
S.code = E.code || gen(top.get(id.lexeme) '=' E.addr)
```
其中 `top.get(id.lexeme)`从符号表中查找 id 的地址

### 增量式翻译
增量式翻译方案：如果code属性知识不断追加生成的指令，就不一定真的构造一个大字符串E.code，而是可以再语义动作发生时直接emit/gen指令

非增量式：
```
E.code = E1.code || E2.code || gen(...)
```
增量式：
```
翻译 E1 时直接 emit
翻译 E2 时直接 emit
最后 emit 当前运算指令
```

### 数组引用翻译：核心是算地址
数组翻译的核心不是“取值”，而是**算出被访问元素的地址或偏移量**
数组引用生成代码的核心是确定数组引用的地址

对于`int a[2][3]`
访问`c+a[i][j]`时
需要算出`a[i][j]`的偏移量
```
a 的类型：array(2, array(3, integer))
int 的宽度为 4
a 的总宽度：2 * 3 * 4 = 24
a[i] 的类型：array(3, integer)
a[i] 的宽度：3 * 4 = 12
a[i][j] 的类型：integer
```

用L表示左值数组引用
```
L → id [ E ]
L.array = top.get(id.lexeme)
L.type  = L.array.type.elem
L.addr  = newTemp()
gen(L.addr '=' E.addr '*' L.type.width)
```

### 类型检查和类型转换
**类型综合**
子表达式推父表达式
```
E1.type = int
E2.type = float
E1 + E2 的 type = float
```

**类型推导**
根据使用方式反推类型
```
如果 f(x) 是表达式
则存在 α, β
使得 f : α → β，x : α
```

类型转换中有两个重要概念
```
widening:拓宽转换 例如 int -> float
narrowing:窄化转换 例如 float -> int
```
隐式转换由编译器自动插入；
显式转换由程序员写出，例如`(float)i`；

类型转换SDT
```
E → E1 + E2 {
    E.type = max(E1.type, E2.type)
    a1 = widen(E1.addr, E1.type, E.type)
    a2 = widen(E2.addr, E2.type, E.type)
    E.addr = newTemp()
    gen(E.addr '=' a1 '+' a2)
}
```

其中max找两个类型在拓宽层次结构中的最小公共祖先，widen负责生成必要转换代码

### 控制流翻译：布尔表达式有两种角色

布尔表达式有两种用法
**改变控制流**
```c
if (x < y) S;
while (x < y) S;
```
这是布尔表达式的值不一定真的保存为true或false，而是通过跳转到不同位置隐含表示

**计算逻辑流**
```c
x = a < b;
y = p && q;
```
这时表达式的结果需要真的放进某个变量或临时变量中

所以编译器会用两套逻辑处理：
```
jump 版本：生成跳转代码
rvalue 版本：生成求值代码
```

例如，
```c
if (a < b) x = 1;
```
可以生成：
```
if a < b goto Ltrue
goto Lfalse
Ltrue:
x = 1
Lfalse:
...
```

而
```c
x = a < b;
```
可能生成
```
if a < b goto Ltrue
goto Lfalse
Ltrue:
t = true
goto Lend
Lfalse:
t = false
Lend:
x = t
```

### 布尔表达式与短路求值
将布尔表达式非终结符记为B，包括
```
B1 || B2
B1 && B2
!B1
E1 rel E2
true
false
```
`&&`和`||`通常采用短路求值
如果`B1||B2`中`B1`为真，整个表达式已经为真，不需要计算B2
如果`B1&&B2`中`B1`为假，整个表达式已经为假，不需要计算B2

### 非回填方案：true、false、next继承属性
为布尔表达式B设置继承属性
```
B.true
B.false
```
为语句S设置继承属性
`S.next`

```
B.true：B 为真时跳到哪里
B.false：B 为假时跳到哪里
S.next：S 执行完后跳到哪里
```

例如：
```
B → E1 rel E2
翻译为：
if E1.addr rel.op E2.addr goto B.true
goto B.false
```

if(B) S1 的规则大致是：
```
B.true  = newlabel()
B.false = S1.next = S.next
S.code  = B.code || label(B.true) || S1.code
```

`newlabel` 生成新标号
`label(x)`把下一条指令的位置绑定到标号x

### fall-through 减少冗余 goto
直接翻译布尔表达式容易产生冗余goto
```
if x>200 goto L4
goto L1
L4:
...
```
如果真分支刚好就是下一条代码，可以改为：
```
ifFalse x > 200 goto L1
L4:
...
```
这样少一条goto

引入特殊标号`fall`，表示“穿越”或 fall-through，即不生成跳转指令，直接顺序执行下一条指令

例如：
```
B.true = fall
B.false = S.next
```

### 为什么要回填backpatching？

控制流代码生成有一个核心问题：生成跳转指令时，目标位置可能还不知道，这是向前引用forward reference
```c
if (B) S
```

当你正在生成B的代码时，S的代码还没有生成完，所以“B为假时应该跳到S后面”这个目标地址未知

解决方法： **回填 backpatching**
```
先生成带占位符的跳转指令：
if a < b goto _
goto _

等目标地址确定后，再把 _ 填成真实地址。
```

回填的好处是可以单遍生成代码，适合语法制导翻译和 LALR(1) 这类分析器。

### 布尔式回填： truelist, falselist

回填方案不再立刻填写 B.true 和 b.false ，而是让 B 有两个综合属性
```
B.truelist
B.falselist

含义
truelist：B 为真时执行的、目标尚未填写的跳转指令列表
falselist：B 为假时执行的、目标尚未填写的跳转指令列表

makelist(i)：创建只包含 i 的列表
merge(p1, p2)：合并两个列表
backpatch(p, i)：把列表 p 中所有跳转指令的目标都填为 i
```
