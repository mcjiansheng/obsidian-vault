# 文件名：编译原理.md（压缩版）

## 1. 本章核心框架
编译器整体流水线，从源代码到目标代码的八个阶段依次串联。

```
源代码 → 词法分析(Token序列) → 语法分析(AST) → 语义分析(类型/符号表)
→ 中间代码生成(IR/三地址码) → 机器无关优化 → 代码生成(目标代码) → 运行时环境
```

- 前端（Front-end）：词法+语法+语义+中间代码生成
- 后端（Back-end）：指令选择+寄存器分配+指令排序

## 2. 必背概念

| 概念 | 英文/缩写 | 一句话定义 |
|------|-----------|-----------|
| 编译器 | Compiler | 把高级语言翻译成等价目标语言的程序 |
| 解释器 | Interpreter | 不生成目标程序，边解释边执行 |
| 词素 | Lexeme | 源代码中具体的字符片段（如 `position`, `60`） |
| 词法单元 | Token | 词素的抽象类别（如 `<id,1>`, `<number,4>`） |
| 终结符 | Terminal | 不可再展开的基本符号（如 `id`, `+`, `if`） |
| 非终结符 | Non-terminal | 可继续展开的语法类别（如 `E`, `Stmt`） |
| 上下文无关文法 | CFG | `G = (V, Σ, P, S)`，产生式左部只能是非终结符 |
| 开始符号 | Start Symbol | 文法推导的起点 |
| 推导 | Derivation | 从开始符号逐步替换为目标串的过程 |
| 语法树 | Parse Tree | 推导过程的树形表示 |
| 二义性 | Ambiguity | 同一句子可生成多棵不同语法树 |

## 3. 各阶段核心任务速查

| 阶段 | 输入 → 输出 | 核心工作 |
|------|-------------|---------|
| 词法分析 | 字符流 → Token序列 | 识别词素，生成Token |
| 语法分析 | Token序列 → 语法树/AST | 验证语法，构建树结构 |
| 语义分析 | AST + 符号表 | 类型检查、作用域检查 |
| 中间代码生成 | AST → 三地址码/IR | 生成平台无关中间表示 |
| 机器无关优化 | IR → 优化后IR | 数据流分析、循环优化等 |
| 代码生成 | IR → 目标代码 | 指令选择、寄存器分配 |
| 运行时环境 | — | 栈/堆管理、函数调用、GC |

## 4. 易错点与速记
- 【易错】编译器≠解释器：编译器生成目标代码后执行，解释器直接执行源程序
- 【易错】词法分析处理「字符→Token」；语法分析处理「Token→AST」；不可混淆层级
- 【考点】CFG 四元组定义：G=(V, Σ, P, S)，必须记住各符号含义
- 【速记】前端=理解源代码，后端=生成目标码

## 5. 打印版极简总结
```
字符流 → 词法(Token) → 语法(AST) → 语义(类型) → IR(三地址码) → 优化 → 目标代码
        └──前端──┘                                    └──后端──┘
```
