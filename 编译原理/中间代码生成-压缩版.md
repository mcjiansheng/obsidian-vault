# 文件名：中间代码生成.md（压缩版）

## 1. 本章核心框架
中间代码生成将 AST 翻译成接近机器但不绑定具体机器的 IR。核心形式是**三地址码(TAC)**，重点是表达式的翻译和**布尔表达式的回填(backpatching)**。

```
AST → 三地址码(TAC) / SSA → 优化器 → 后端代码生成
       ↑ 核心：code/addr属性 + 回填(backpatching)
```

## 2. 必背概念

- 【定义】IR Intermediate Representation：前端与后端之间独立的程序表示
- 【定义】三地址码 TAC：`x = y op z`，每条指令右边最多一个运算符
- 【定义】四元式 Quadruple：`(op, arg1, arg2, result)`，如 `(+, y, z, x)`
- 【定义】三元式 Triple：`(op, arg1, arg2)`，用位置编号表示结果
- 【定义】间接三元式：用指针表指向三元式，优化时只移动指针
- 【定义】SSA Static Single Assignment：每个变量版本只赋值一次
- 【定义】φ 函数：`x3 = φ(x1, x2)`，根据来路选择值（IR 伪指令）
- 【定义】短路求值 Short-circuit：`B1||B2` 若 B1 为真不执行 B2；`B1&&B2` 若 B1 为假不执行 B2
- 【定义】回填 Backpatching：先生成带占位符的跳转指令，等目标确定后再填入
- 【定义】truelist / falselist：B 为真/假时执行的目标尚未确定的跳转指令列表
- 【定义】nextlist：语句执行完后应跳转到的、目标尚未确定的指令列表
- 【定义】fall-through（穿越）：不生成跳转，直接顺序执行下一条

## 3. 关键算法 / 规则

### 3.1 表达式翻译：code + addr

【算法】对 `E → E₁ + E₂`：
```
E.addr = newTemp()
E.code = E₁.code || E₂.code || gen(E.addr '=' E₁.addr '+' E₂.addr)
```

【公式】`newTemp()` 生成临时变量；`gen(...)` 生成一条三地址指令

### 3.2 三地址码主要指令

```
x = y op z        // 算术/逻辑
x = y             // 复制
goto L            // 无条件跳转
if x relop y goto L  // 条件跳转
param x; call f,n // 函数调用
x = a[i]; a[i] = y  // 数组访问
```

### 3.3 数组引用翻译

【算法】核心是算地址偏移：
```
L → id [E]
L.addr = newTemp()
gen(L.addr '=' E.addr '*' L.type.width)
```
对多维数组如 `a[i][j]`：`offset = i*row_width + j*elem_width`

### 3.4 布尔表达式回填

【算法】辅助函数：
```
makelist(i)     → 创建只含指令 i 的列表
merge(p1, p2)   → 合并两个列表
backpatch(p, i) → 把列表 p 中所有跳转目标填为 i
```

### 3.5 `||` 和 `&&` 的回填规则

【算法】
```
B → B₁ || M B₂:
    backpatch(B₁.falselist, M.instr)     // B₁为假才跳到B₂
    B.truelist  = merge(B₁.truelist, B₂.truelist)
    B.falselist = B₂.falselist

B → B₁ && M B₂:
    backpatch(B₁.truelist, M.instr)      // B₁为真才跳到B₂
    B.truelist  = B₂.truelist
    B.falselist = merge(B₁.falselist, B₂.falselist)

B → ! B₁:
    B.truelist  = B₁.falselist           // 真假互换
    B.falselist = B₁.truelist
```

### 3.6 控制流语句回填

【算法】关键标记非终结符：
```
M → ε    { M.instr = nextinstr }          // 记录下一条指令位置
N → ε    { N.nextlist = makelist(nextinstr); emit("goto _") }  // 生成跳转占位

S → if (B) M₁ S₁ N else M₂ S₂:
    backpatch(B.truelist,  M₁.instr)       // B为真→S₁开始
    backpatch(B.falselist, M₂.instr)       // B为假→S₂开始
    S.nextlist = merge(S₁.nextlist, N.nextlist, S₂.nextlist)

S → while M₁ (B) M₂ S₁:
    backpatch(S₁.nextlist, M₁.instr)       // 循环体结束→回到条件判断
    backpatch(B.truelist,  M₂.instr)       // B为真→进入循环体
    S.nextlist = B.falselist               // B为假→跳出循环
```

### 3.7 类型转换 SDT

【算法】
```
E → E₁ + E₂:
    E.type = max(E₁.type, E₂.type)
    a₁ = widen(E₁.addr, E₁.type, E.type)
    a₂ = widen(E₂.addr, E₂.type, E.type)
    E.addr = newTemp()
    gen(E.addr '=' a₁ '+' a₂)
```
max 取拓宽层次中的最小公共祖先

## 4. 典型题型与解题模板

### 题型1：表达式 → 三地址码
- 识别特征：给中缀表达式，要求翻译成 TAC
- 解题步骤：①按优先级拆分子表达式 ②为每个子表达式引入临时变量 ③按序生成指令
- 示例：`x = a+b*c` → `t1=b*c; t2=a+t1; x=t2`

### 题型2：布尔表达式回填
- 识别特征：给 `if (B) S` 或 `while (B) S`，要求生成回填代码
- 解题步骤：①生成 B 的跳转代码（占位符）②生成 S 的代码 ③回填目标地址 ④合并 nextlist
- 常见陷阱：忘记录 M/N 的位置；漏掉 merge

### 题型3：数组引用翻译
- 识别特征：`a[i][j]` 类数组访问
- 解题步骤：①确定元素类型宽度 ②计算多维偏移 ③生成地址计算和 load/store 指令
- 常见陷阱：忘记乘元素宽度

## 5. 易错点与速记

- 【易错】四元式有 result 字段，三元式用位置编号表示结果；间接三元式用指针表避免编号变动
- 【易错】`||` 回填 falselist；`&&` 回填 truelist — 两者正好相反
- 【易错】`!` 把 truelist 和 falselist 互换
- 【易错】回填的本质是处理向前引用(forward reference)：目标地址尚未确定，先留占位符
- 【考点】SSA 的 φ 函数在控制流汇合处合并不同来源的值
- 【速记】短路求值 = `||`(真即停) + `&&`(假即停)

## 6. 打印版极简总结
```
表达式翻译：code + addr + newTemp + gen → TAC序列
数组翻译：地址 = 基址 + 下标 × 元素宽度
布尔表达式：回填 = truelist + falselist + makelist/merge/backpatch
控制流回填：M(记录位置) + N(生成占位goto) + nextlist(合并出口)
类型转换：max(类型) + widen/narrow
```
