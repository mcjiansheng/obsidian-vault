[[编译原理]]
词法分析将源代码字符流切成了Token序列：
```c
position = initial + rate * 60
```
词法分析得到：

```
<id,1> <=,> <id,2> <+,> <id,3> <*,> <number,4>
```

语法分析器的任务就是判断该 Token序列是否符合语言的语法规则，并构造出语法结构

类型检查、语义分析、中间代码生成等有时也会和语法分析交错完成

```
词法分析：字符流 -> token序列
语法分析：token序列 -> 语法树 / AST
语义分析：语法树 + 符号表 -> 类型检查、作用域检查
```

语法分析不仅要知道Token串是否合法，还需要知道它如何被理解

这涉及到 **文法、推导、语法树、二义性、优先级、结合性** 等概念
