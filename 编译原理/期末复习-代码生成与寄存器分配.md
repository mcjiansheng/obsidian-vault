按题目给出的机器格式，`OP` 的两个源操作数应在寄存器中，所以 `a*b` 要先把 `a`、`b` 都装入寄存器。

```asm
// (1) t = a * b
LD  R1, a
LD  R2, b
MUL R1, R1, R2      // R1 = a * b，即 t

// (2) u = x + t
LD  R2, x           // R2 原来放 b，但 b 的内存中已有正确值，可直接覆盖
ADD R2, R2, R1      // R2 = x + t，即 u

// (3) x = u
// 不需要生成机器指令，只更新描述符：x 和 u 的当前值都在 R2 中

// (4) 结束时写回内存
ST  t, R1
ST  u, R2
ST  x, R2
```
- `R1 / R2` 列是**寄存器描述符**：该寄存器中当前保存哪些变量的最新值。
- `x / a / b / t / u` 列是**地址描述符**：该变量当前值存在于哪些位置。
#### （1）`t = a * b`
```
LD  R1, a
LD  R2, b
MUL R1, R1, R2
```

|R1|R2|x|a|b|t|u|
|---|---|---|---|---|---|---|
|t|b|x|a|b, R2|R1|u|
#### （2）`u = x + t`
```
LD  R2, x
ADD R2, R2, R1
```

| R1  | R2  | x   | a   | b   | t   | u   |
| --- | --- | --- | --- | --- | --- | --- |
| t   | u   | x   | a   | b   | R1  | R2  |
#### （3）`x = u`
不生成机器代码，只更新描述符。

|R1|R2|x|a|b|t|u|
|---|---|---|---|---|---|---|
|t|u, x|R2|a|b|R1|R2|
#### （4）结束时写回内存
```
ST t, R1
ST u, R2
ST x, R2
```

| R1  | R2   | x     | a   | b   | t     | u     |
| --- | ---- | ----- | --- | --- | ----- | ----- |
| t   | u, x | x, R2 | a   | b   | t, R1 | u, R2 |
