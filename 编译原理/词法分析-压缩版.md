
## 1. 本章核心框架
词法分析器读入字符流，识别词素(lexeme)，输出词法单元(token)序列。核心工具链：
```
正则表达式(描述模式) → NFA(Thompson构造) → DFA(子集构造) → 最小化DFA → 词法分析器代码
```

## 2. 必背概念

- 【定义】词素 Lexeme：源代码里真实出现的字符序列（如 `position`, `60`, `+`）
- 【定义】词法单元 Token：`<token-name, attribute-value>` 对（如 `<id,1>`, `<number,4>`）
- 【定义】模式 Pattern：描述某类词素长什么样的规则
- 【定义】正则表达式 RE：描述 Token 模式的形式化工具
- 【定义】语言 Language：某个字母表上串的集合
- 【定义】ε：空串，长度为 0
- 【定义】NFA 非确定有穷自动机：同一状态读同一字符可走多条边；可有 ε 边
- 【定义】DFA 确定有穷自动机：无 ε 边；每个状态对每个输入有且仅有一条出边
- 【定义】ε-closure(T)：从 T 出发只沿 ε 边可达的所有状态 + T 自身
- 【定义】move(T, a)：从 T 中任意状态读入 a 后能到达的状态集合

## 3. 关键算法 / 规则

### 3.1 正则表达式运算
| 运算 | 记法 | 含义 |
|------|------|------|
| 并 | `r\|s` | L(r) ∪ L(s) |
| 连接 | `rs` | L(r)L(s) |
| Kleene闭包 | `r*` | 重复 0 次或多次（含 ε） |
| 正闭包 | `r+` | 重复 1 次或多次（等价 `rr*`） |
| 可选 | `r?` | 0 次或 1 次（等价 `ε\|r`） |

优先级：`* > 连接 > |`

### 3.2 Thompson构造：正则表达式 → NFA

【算法】
1. **选择 `s|t`**：新建开始状态 → ε 分别连到 N(s) 和 N(t) 的开始状态；两者的接受状态 → ε 连到新接受状态
2. **连接 `st`**：N(s) 的接受状态 → ε 连到 N(t) 的开始状态；整体开始=N(s)开始，整体接受=N(t)接受
3. **闭包 `s*`**：新建开始和接受状态；开始 → ε 直接到接受（0次）；开始 → ε 到 N(s)；N(s) 接受 → ε 回 N(s) 开始（多次）；N(s) 接受 → ε 到新接受（结束）

### 3.3 子集构造法：NFA → DFA

【算法】
1. DFA 开始状态 = `ε-closure({s0})`
2. 对每个 DFA 状态 T 和每个输入字符 a：
   - `U = ε-closure(move(T, a))`
   - 若 U 是新集合，加入 DFA 状态集
   - 添加转换 `T --a--> U`
3. DFA 接受状态 = 所有包含 NFA 接受状态的状态集合

【考点】DFA 每个状态对应 NFA 一个状态集合；最坏情况下 DFA 状态数是指数级，但实践中通常相近

### 3.4 DFA 最小化

【算法】
1. 初始化：`Π = {S-F, F}`（非接受状态集合 + 接受状态集合）
2. 不断检查每组：若组内某些状态读同一字符后跳到不同组，则拆分
3. 重复直到不再变化

### 3.5 最长匹配原则 Maximal Munch
从当前位置尽可能读入最多字符，直到再读就不能匹配任何规则；回退到最后一次能接受的位置，输出对应 token。

### 3.6 规则优先级
多个规则匹配长度相同时，按规则定义顺序，先出现的优先（如关键字 `"if"` 优先于标识符规则）。

## 4. 典型题型与解题模板

### 题型1：正则表达式写Token模式
- 识别特征：给出某类词素描述，要求写正则表达式
- 解题步骤：①识别基本字符类（字母、数字等）②用连接组合 ③用 `|`、`*`、`+` 表达选择/重复
- 示例：标识符 → `[A-Za-z_][A-Za-z0-9_]*`

### 题型2：正则表达式 → NFA（Thompson构造）
- 识别特征：给出正则表达式，要求画 NFA
- 解题步骤：按表达式结构递归构造（先拆成子表达式，再套用三种核心模式）
- 常见陷阱：连接和选择的 ε 边容易漏画

### 题型3：NFA → DFA（子集构造）
- 识别特征：给出 NFA 状态转换表/图，要求构造等价 DFA
- 解题步骤：①从开始状态的 ε-closure 出发 ②逐字符计算 move + ε-closure ③每个新集合作为一个 DFA 状态 ④标记接受状态
- 常见陷阱：忘记对新状态也要求 ε-closure；接受状态判断遗漏

### 题型4：DFA 最小化
- 识别特征：给出 DFA，要求最小化
- 解题步骤：①初始划分：接受/非接受 ②逐字符检查组内一致性 ③拆分不一致的状态 ④重复至稳定
- 常见陷阱：拆分后忘记重新检查之前已检查过的组

## 5. 易错点与速记

- 【易错】`ε-closure(T)` 包含 T 自身，不仅仅是沿 ε 边到达的状态
- 【易错】NFA 接受条件是"存在一条路径"到达接受状态，不是所有路径
- 【易错】关键字和标识符冲突：先按标识符识别，再查关键字表区分
- 【易错】带 `*` 的接受状态：最后读入的符号不属于当前词素，需回退 forward 指针
- 【考点】NFA 和 DFA 表达力等价，都能识别正则语言
- 【速记】正则→NFA→DFA→最小化DFA→词法分析器代码

## 6. 打印版极简总结
```
词法分析 = 字符流 → 正则匹配 → Token序列
核心：正则表达式(描述) → NFA(Thompson) → DFA(子集构造) → 最小化 → 代码
原则：最长匹配 + 规则优先级
关键操作：ε-closure(T), move(T,a)
```
