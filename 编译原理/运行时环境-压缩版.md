# 文件名：运行时环境.md（压缩版）

## 1. 本章核心框架
运行时环境解决编译器如何为源语言中的抽象概念在真实机器上安排存储、调用、访问和回收机制。

```
存储布局：代码区 | 静态区 | 堆区(↑) | ... | 栈区(↓)
          ↑编译时确定  ↑全局/静态  ↑动态分配    ↑函数调用
          
核心问题：数据放哪里？函数怎么调？嵌套作用域怎么访问？堆对象怎么回收？
```

## 2. 必背概念

### 2.1 存储区域
- 【定义】代码区 Code Area：存放目标程序机器指令，编译时确定大小
- 【定义】静态区 Static Area：全局变量、静态变量、常量 → 生命周期=整个程序
- 【定义】栈区 Stack：活动记录/栈帧 → 后进先出，与函数调用同步
- 【定义】堆区 Heap：生命周期不固定 → malloc/new 对象，可能比创建者活得更久

### 2.2 活动与栈帧
- 【定义】活动 Activation：一次函数调用
- 【定义】活动树 Activation Tree：节点=活动，父节点调用子节点，前序遍历=调用序，后序遍历=返回序
- 【定义】活动记录/栈帧 Activation Record/Stack Frame：一次函数调用所需的运行时存储块
- 【定义】控制链 Control Link：指向调用者的活动记录（"谁调用了我"）
- 【定义】访问链 Access Link：指向静态外层作用域的活动记录（"我的外层是谁"）
- 【定义】嵌套深度 Nesting Depth：不嵌套=1；嵌套在深度 i 的过程中=i+1

### 2.3 寄存器
- 【定义】eip：指向代码区下一条要执行的指令
- 【定义】ebp：指向当前栈帧的栈底（帧基址）
- 【定义】esp：指向当前栈顶（随压栈退栈变化）

### 2.4 栈帧字段
```
┌──────────────┐
│ 实在参数      │  ← 调用者放，被调用者活动记录开始位置
│ 返回值        │
│ 控制链        │  ← 固定长度字段（中间）
│ 访问链        │
│ 机器状态      │
│ 局部数据      │
│ 临时值        │
│ 变长数组      │  ← 编译时大小未知，放尾部
└──────────────┘
```

### 2.5 堆与GC
- 【定义】存储管理器 Memory Manager：负责堆空间的分配与回收
- 【定义】Best-Fit：选能满足请求的最小空闲块（保留大块，但碎片多）
- 【定义】First-Fit：选第一个能容纳的空闲块（快，局部性好）
- 【定义】边界标记 Boundary Tag：块两端设 free/used 位+大小，用于合并相邻空闲块
- 【定义】根集 Root Set：不需指针解引用就能访问的数据（静态成员、栈中局部变量等）
- 【定义】可达性 Reachability：根集对象可达；可达对象引用的对象也可达
- 【定义】增变者 Mutator：用户程序，不断修改堆对象集合

## 3. 关键算法 / 规则

### 3.1 访问链访问非局部变量

【算法】当前深度 np，变量深度 nq：
1. 从当前活动记录出发，沿访问链走 `np - nq` 步
2. 找到目标活动记录后，用变量在其中的固定偏移访问

### 3.2 访问链设置规则

【算法】当 q 调用 p 时：
- **p 深度 > q**（p 定义在 q 内部）：`p.access_link = q 的当前活动记录`
- **p = q**（递归）：`new.access_link = old.access_link`
- **p 深度 ≤ q**：找到 r（p 定义在 r 中且 q 嵌套在 r 中）；`p.access_link = 栈中最高的 r 活动记录`

### 3.3 显示表 Display

【定义】数组 `d[i]` 指向栈中嵌套深度为 i 的最高活动记录

【算法】访问深度 i 的外层变量：直接用 `d[i] + offset`（O(1) 而非沿链走）
```
调用 p 时：保存旧 d[np]; d[np] = 当前活动记录
返回时：恢复旧 d[np]
```

对比：
| | 访问链 | 显示表 |
|------|--------|--------|
| 结构 | 链表 | 数组 |
| 访问 | 沿链走 np-nq 步 | O(1) 直接索引 |
| 维护 | 调用时设链 | 调用/返回时维护数组 |

### 3.4 堆分配方法

| 方法 | 策略 | 优点 | 缺点 |
|------|------|------|------|
| Best-Fit | 选最小满足的空闲块 | 保留大块 | 碎片多、查找慢 |
| First-Fit | 选第一个能容纳的块 | 快、局部性好 | 空间利用可能差 |
| Segregated Free List | 同规格块放同一容器 | 高效、减少碎片 | 实现复杂 |

### 3.5 垃圾回收算法对比

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 引用计数 | 对象维护引用计数，=0 回收 | 即时回收，无停顿 | 无法处理循环引用，开销大 |
| 标记-清扫 Mark-Sweep | 标记可达→清扫不可达 | 简单，不移动对象 | 碎片、Stop-the-world |
| 标记-压缩 Mark-Compact | 标记→移到一端→更新引用 | 消除碎片 | 移动成本高 |
| 拷贝回收 Copying | From-space→To-space 复制可达 | 无碎片、分配快 | 一半空间浪费 |
| 分代回收 Generational | 年轻代频繁回收，老年代低频 | 利用"朝生夕死" | 需记忆集处理跨代引用 |

### 3.6 拷贝回收算法

【算法】
1. 程序在 From-space 分配
2. From-space 满 → GC 启动
3. 从根集出发，可达对象复制到 To-space
4. 更新所有引用
5. From-space 整体视为空闲
6. 两半空间交换身份

### 3.7 分代回收与记忆集

【原理】大多数对象朝生夕死 → 频繁回收年轻代收益高

【记忆集 Remembered Set】记录老年代对象对年轻代对象的引用 → 只扫描年轻代时不会漏掉跨代引用

## 4. 典型题型与解题模板

### 题型1：活动树绘制
- 识别特征：给函数调用序列，画活动树
- 解题步骤：①main 为根 ②按调用顺序画子节点 ③前序遍历=调用序，后序遍历=返回序

### 题型2：栈帧布局
- 识别特征：给函数声明（含参数、局部变量、变长数组），要求画栈帧
- 解题步骤：①参数/返回值在开头 ②控制链/访问链/机器状态在中间 ③定长局部变量 ④变长数据在尾部（运行时计算偏移）
- 常见陷阱：变长数组不能像普通变量一样用固定偏移访问

### 题型3：嵌套过程非局部变量访问
- 识别特征：给嵌套过程代码，问"B 中如何访问 A 的变量 x"
- 解题步骤：①计算嵌套深度 np 和 nq ②np-nq=沿访问链步数 ③按偏移访问
- 常见陷阱：若使用访问链，步数=嵌套深度差

### 题型4：访问链/显示表维护
- 识别特征：给嵌套过程调用序列，要求画出各活动记录的访问链指向
- 解题步骤：①确定调用时各过程的嵌套深度 ②按三条规则设置访问链（p>q/p=q/p≤q）

### 题型5：GC 可达性分析
- 识别特征：给堆对象引用图 + 根集，判断可达/不可达
- 解题步骤：①从根集出发 DFS/BFS 标记可达 ②未被标记的不可达
- 常见陷阱：循环引用但无外部引用 → 在引用计数中为 1 但不可达

### 题型6：堆分配模拟
- 识别特征：给空闲块列表 + 分配/释放请求序列
- 解题步骤：①按策略（Best/First Fit）选块 ②释放时合并相邻空闲块

## 5. 易错点与速记

- 【易错】控制链="谁调用了我"（动态）；访问链="我在谁的词法作用域中"（静态）——两者不同
- 【易错】C 语言无嵌套过程，只有控制链无需访问链；Pascal 类语言才需要访问链
- 【易错】过程参数作为参数传递时，需要传递代码指针 + 访问链（闭包雏形）
- 【易错】变长数据在栈顶之上，偏移需运行时计算（`top_sp` 用于定长字段，`top` 用于实际栈顶）
- 【易错】栈式分配支持递归（每次调用新建活动记录），静态分配不支持
- 【易错】C/C++ 类型不安全 → 不适合自动 GC
- 【易错】参数入栈顺序在 C 中与书写顺序相反（从右到左）
- 【考点】GC 一般只能回收不可达对象（狭义垃圾），不能回收"之后不再使用但可达"的对象（广义垃圾）
- 【速记】活动记录 = 参数+返回值+控制链+访问链+机器状态+局部数据+临时值
- 【速记】存储四区：代码(指令) + 静态(全局) + 栈(函数调用LIFO) + 堆(动态/GC)

## 6. 打印版极简总结
```
存储四区：代码 | 静态 | 栈(LIFO,函数调用) | 堆(动态,GC)
栈帧 = 参数 + 返回值 + 控制链(谁调用我) + 访问链(我的外层) + 局部变量 + 变长区
访问链 vs 显示表：链表(O(n)) vs 数组(O(1))
GC五法：引用计数/标记清扫/标记压缩/拷贝回收/分代回收
分代核心：朝生夕死 + 记忆集
```
