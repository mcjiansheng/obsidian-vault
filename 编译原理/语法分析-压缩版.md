# 文件名：语法分析.md（压缩版）

## 1. 本章核心框架
语法分析器读入 Token 序列，验证语法规则，构造语法树/AST。核心是自顶向下分析（LL）和自底向上分析（LR）两大范式。

```
Token序列 → 预测分析/递归下降 → 语法树 → AST
          ↑ 需要LL(1)文法：无左递归 + 无左公因子
```

## 2. 必背概念

- 【定义】文法 Grammar：描述语法的形式化工具，CFG `G = (V, Σ, P, S)`
- 【定义】上下文无关文法 CFG：产生式左部只能是单个非终结符；推导不依赖上下文
- 【定义】终结符 Terminal：不可再展开的基本符号（对应 Token，如 `id`, `+`, `*`, `if`）
- 【定义】非终结符 Non-terminal：可继续展开的语法类别（如 `E`, `T`, `F`, `Stmt`）
- 【定义】推导 Derivation：从开始符号不断替换非终结符的过程
- 【定义】句型 Sentential Form：推导过程中间出现的串（可含非终结符）
- 【定义】句子 Sentence：只含终结符的句型
- 【定义】语言 Language：文法能生成的所有句子的集合，记 L(G)
- 【定义】最左推导：每次都替换最左边的非终结符
- 【定义】最右推导：每次都替换最右边的非终结符
- 【定义】二义性 Ambiguity：同一句子有多棵语法树
- 【定义】CST 具体语法树：保留完整推导细节
- 【定义】AST 抽象语法树：去掉冗余语法，只保留语义结构
- 【定义】LL(1)：从左到右扫描，最左推导，向前看 1 个符号
- 【定义】FIRST(α)：α 能推导出的串的首终结符集合（含 ε 如果 α⇒*ε）
- 【定义】FOLLOW(A)：在某些句型中紧跟在 A 右边的终结符集合（含 $ 如果 A 可出现在末尾）
- 【定义】SELECT(A→α)：当前输入符号为什么时可选择该产生式

## 3. 关键算法 / 规则 / 公式

### 3.1 消除左递归

【算法】若有 `A → Aα1 | Aα2 | β1 | β2`（β 不以 A 开头），改写为：
```
A  → β1 A' | β2 A'
A' → α1 A' | α2 A' | ε
```

【例题模板】
```
E → E + T | T
⇒ E → T E'
   E' → + T E' | ε
```

【考点】间接左递归：先排序、替换、转直接左递归、再消除

### 3.2 提取左公因子

【算法】若有 `A → αβ1 | αβ2`，改写为：
```
A  → α A'
A' → β1 | β2
```

### 3.3 FIRST 集计算

【算法】
1. 对终结符 a：`FIRST(a) = {a}`
2. 对非终结符 A：若有 `A → X1 X2 ... Xk`
   - 把 `FIRST(X1) - {ε}` 加入 FIRST(A)
   - 若 X1 能推出 ε，继续看 X2，以此类推
   - 若所有 Xi 都能推出 ε，则 ε ∈ FIRST(A)

### 3.4 FOLLOW 集计算

【算法】
1. `$ ∈ FOLLOW(S)`（S 为开始符号）
2. 若 `A → α B β`：`FIRST(β) - {ε}` 加入 FOLLOW(B)
3. 若 `A → α B β` 且 `β ⇒* ε`（或 `A → α B`）：FOLLOW(A) 加入 FOLLOW(B)

### 3.5 SELECT 集与 LL(1) 判断

【公式】
```
SELECT(A → α) = FIRST(α)              若 ε ∉ FIRST(α)
SELECT(A → α) = FIRST(α) ∪ FOLLOW(A)  若 ε ∈ FIRST(α)
```

【考点】LL(1) 判别条件：同一非终结符的各候选式的 SELECT 集两两不相交

### 3.6 LL(1) 预测分析表构造

【算法】对每条产生式 `A → α`：
1. 对所有 `a ∈ FIRST(α) - {ε}`，填 `M[A, a] = A → α`
2. 若 `ε ∈ FIRST(α)`，对所有 `b ∈ FOLLOW(A)`，填 `M[A, b] = A → α`
3. 表项冲突 ⇒ 文法不是 LL(1)

### 3.7 预测分析流程

【算法】
1. 预处理文法：消除左递归 + 提取左公因子
2. 构造预测分析表 M
3. 用栈分析：栈顶非终结符 A + 当前输入 a → 查 M[A,a] → 展开/匹配

## 4. 典型题型与解题模板

### 题型1：消除左递归 + 提取左公因子
- 识别特征：文法含 `A → Aα` 或 `A → αβ1 | αβ2`
- 解题步骤：①先消直接左递归（套公式）②检查间接左递归 ③提取左公因子
- 常见陷阱：忘记处理 ε

### 题型2：求 FIRST/FOLLOW 集
- 识别特征：给出文法，求各非终结符的 FIRST 和 FOLLOW
- 解题步骤：①FIRST：自底向上，先终结符后非终结符 ②FOLLOW：迭代计算，注意 $ 和 β⇒*ε 的情况
- 常见陷阱：FOLLOW 要反复迭代直到不动点；ε 的处理

### 题型3：判断是否为 LL(1) 文法
- 识别特征：给出文法，要求判断
- 解题步骤：①消左递归、提左公因子 ②求 FIRST/FOLLOW ③求 SELECT ④检查 SELECT 是否两两不相交
- 常见陷阱：忘记预处理就直接判断

### 题型4：构造 LL(1) 预测分析表
- 识别特征：给出 LL(1) 文法，构造预测分析表
- 解题步骤：①求 FIRST/FOLLOW ②按 SELECT 填表
- 常见陷阱：ε ∈ FIRST 时，要对 FOLLOW(A) 中所有符号填表

### 题型5：用预测分析表分析输入串
- 识别特征：给分析表和输入串，模拟分析过程
- 解题步骤：①栈初始为 `$S` ②根据栈顶和输入符号查表 ③匹配/展开直到栈空

## 5. 易错点与速记

- 【易错】FIRST(α) 是首终结符集合，ε 也是可能的元素；FOLLOW(A) 不含 ε
- 【易错】$ ∈ FOLLOW(S) 总是成立（S 为开始符号）
- 【易错】左递归让递归下降陷入死循环，必须先消除
- 【易错】dangling else：else 匹配最近的尚未匹配的 if
- 【速记】LL(1) = 从左到右 + 最左推导 + 向前看1个符号
- 【速记】正则 < CFG：每个正则语言可用 CFG 描述，反之不成立
- 【速记】SELECT 交集为空 ⇒ LL(1)

## 6. 打印版极简总结
```
自顶向下分析 = 消除左递归 + 提取左公因子 → 求 FIRST/FOLLOW → 求 SELECT
→ 检查 SELECT 互不相交 → 构造预测分析表 → 用栈模拟分析
核心：FIRST(首终结符) / FOLLOW(后面跟什么) / SELECT(何时选此产生式)
```
