# 文件名：代码生成.md（压缩版）

## 1. 本章核心框架
代码生成将优化后的 IR 翻译成目标机器代码。涉及三个互相影响的问题：指令选择、寄存器分配和指派、指令排序。

```
优化后IR → 指令选择 → 寄存器分配 → 指令排序 → 目标代码
              ↑                ↑
         朴素翻译+优化    局部/全局(图染色)
```

## 2. 必背概念

- 【定义】指令选择 Instruction Selection：用哪些机器指令实现 IR 语句
- 【定义】寄存器分配 Register Allocation：哪些值放入寄存器
- 【定义】寄存器指派 Register Assignment：具体放入哪个物理寄存器
- 【定义】基本块 Basic Block：①只能从第一条进入 ②除最后一条外无跳转 ③顺序执行
- 【定义】活跃变量 Live Variable：在某个程序点之后还会使用（且使用前未重新赋值）
- 【定义】待用信息 Next-use：变量下一次会在哪条语句被使用
- 【定义】寄存器描述符 Register Descriptor：记录每个寄存器当前保存了哪些变量/临时值
- 【定义】地址描述符 Address Descriptor：记录每个变量的当前值在哪里（寄存器/内存/两者）
- 【定义】冲突图 Interference Graph：节点=变量/临时值，边=两个值生命期冲突（不能共用寄存器）
- 【定义】图染色 Graph Coloring：用 k 种颜色（k 个寄存器）给冲突图染色
- 【定义】溢出 Spill：寄存器不够时，把部分变量放入内存（增加 load/store）
- 【定义】Chaitin 算法：经典的图染色寄存器分配算法

## 3. 目标机器模型与寻址模式

### 3.1 目标机器指令
```
LD  dst, addr       // 加载
ST  x, r            // 保存
OP  dst, src1, src2 // 计算
BR  L               // 无条件跳转
Bcond r, L          // 条件跳转
```

### 3.2 寻址模式
| 模式 | 含义 | 用途 |
|------|------|------|
| `x` | 变量 x 的内存位置 | 全局/静态变量 |
| `a(r)` | `a + contents(r)` | 数组访问 |
| `constant(r)` | `contents(r) + constant` | 栈帧局部变量 |
| `*r` | 寄存器内容为地址，间接访问 | 指针解引用 |
| `#constant` | 立即数 | 常量 |

### 3.3 静态分配 vs 栈式分配

| | 静态分配 | 栈式分配 |
|------|---------|---------|
| 地址 | `staticArea + offset` | `offset(SP)` 或 `offset(FP)` |
| 优点 | 简单、访问快 | 支持递归、多实例 |
| 缺点 | 不支持递归 | 需要运行时计算地址 |
| 适用于 | 全局变量、静态变量 | 局部变量、函数调用 |

### 3.4 过程调用/返回代码序列

【算法】调用：
```
ADD  SP, SP, #caller.recordSize    // 调整栈指针
ST   0(SP), #here+16               // 保存返回地址
BR   callee.codeArea               // 跳转到被调用过程
```

返回：
```
BR   *0(SP)                        // 跳回返回地址
SUB  SP, SP, #caller.recordSize    // 恢复栈指针
```

## 4. 关键算法

### 4.1 基本块划分

【算法】找入口语句 leader：
1. 程序第一条语句
2. 跳转语句的目标
3. 紧跟在跳转语句之后的语句
两个 leader 之间的语句构成一个基本块

### 4.2 DAG 表示基本块

【算法】用 DAG 表示基本块内表达式，作用：
- 发现公共子表达式（同构节点合并）
- 删除无用赋值
- 减少临时变量
- 重排计算顺序

### 4.3 基本代码生成算法 (getReg)

【算法】对 `x = y op z`：
1. 调用 `getReg(x = y op z)` 选择目标寄存器 R
2. 确保 y 在寄存器中（必要时 LD）
3. 确保 z 可作为操作数
4. 生成 `OP R, Ry, z`
5. 更新寄存器描述符和地址描述符
6. 若 x 在基本块出口仍活跃且只在寄存器中 → 写回内存

**getReg 选择策略**：优先级从高到低：
1. y 已在某寄存器且之后不再使用 → 复用该寄存器
2. 有空寄存器 → 使用空寄存器
3. 选择寄存器腾出：优先选变量已死的；或待用最远的；或内存已有最新副本的

### 4.4 复制语句优化

【算法】对 `x = y`：不生成机器指令，只更新描述符让 x 和 y 共享同一寄存器/位置。

### 4.5 全局寄存器分配：Chaitin 图染色算法

【算法】
1. **构建冲突图**：节点=变量/临时值；边=同时活跃
2. **简化**：找到度数 < k 的节点，删除并压栈
3. **重复**直到图为空；若找不到则选溢出候选
4. **选择**：从栈中弹出节点，分配未被邻居使用的颜色
5. **溢出**：插入 load/store 后重新构图染色

> 度数 < k 的节点总可染色：放回时最多 k-1 个邻居已染色

### 4.6 基本块内优化速查

| 优化 | 示例 | 条件 |
|------|------|------|
| 公共子表达式消除 | `t1=b+c; a=t1; d=t1` | 分量未修改 |
| 死代码删除 | `x=y+z; x=a+b` → 删第一条 | x 后续不用 + 无副作用 |
| 代数化简 | `x=y+0 → x=y` | 浮点需谨慎 |
| 常量折叠 | `x=2*3 → x=6` | — |
| 复制传播 | `x=y; z=x+1 → z=y+1` | x,y 未修改 |

## 5. 典型题型与解题模板

### 题型1：三地址码 → 目标代码（朴素翻译）
- 识别特征：给 TAC 序列，生成简单目标代码
- 解题步骤：①每条 `x=y op z` → `LD R,y; OP R,R,z; ST x,R` ②后续优化
- 常见陷阱：忘记 ST 写回；寄存器复用可省指令

### 题型2：DAG 构造与优化
- 识别特征：给基本块内的 TAC，构造 DAG 并优化
- 解题步骤：①逐条构造 DAG（同表达式合并节点）②从 DAG 重写代码 ③删除死节点
- 关键：DAG 可以自动发现公共子表达式

### 题型3：寄存器描述符/地址描述符模拟
- 识别特征：给 TAC 序列和寄存器数，模拟 getReg 过程
- 解题步骤：①初始化描述符 ②逐条处理 ③维护描述符状态 ④出口写回活跃变量
- 常见陷阱：描述符更新不及时；出口忘记写回

### 题型4：图染色寄存器分配
- 识别特征：给变量活跃区间，构造冲突图并染色
- 解题步骤：①画活跃区间 ②构造冲突图（重叠=边）③简化-染色（Chaitin）④判断是否需要溢出

## 6. 易错点与速记

- 【易错】寄存器描述符记录"寄存器里有什么"；地址描述符记录"变量值在哪里"——两者不同
- 【易错】基本块出口：活跃变量若只在寄存器中必须写回内存，否则后继块读到旧值
- 【易错】数组/指针访问可能产生别名，代码生成器需保守处理
- 【易错】`x = y` 不一定生成 move 指令，可以通过描述符共享位置（零成本复制）
- 【易错】局部寄存器分配在每个基本块出口可能频繁写回，循环性能差
- 【考点】度数 < k 的节点一定能染色，这是 Chaitin 算法正确性的保证
- 【速记】指令选择 = 代码质量的核心；寄存器分配 = 性能的关键；两者互相影响

## 7. 打印版极简总结
```
目标机器模型：LD/ST/OP/BR/Bcond + 6种寻址模式
基本块 = leader划分 → DAG优化(公共子表达式/死代码)
代码生成 = getReg(选择寄存器) + 描述符维护 + 出口写回
寄存器分配 = 局部(基本块内) vs 全局(Chaitin图染色)
图染色 = 冲突图构造 → 简化(度<k) → 选择(着色) → 溢出
```
