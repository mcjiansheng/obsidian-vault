#### 1. 找基本块入口语句
基本块入口语句规则：
1. 程序第一条语句是入口；
2. 任意跳转语句的目标语句是入口；
3. 任意跳转语句的下一条语句是入口。

原程序跳转语句有：
```
(4)  if x < z goto (10)
(7)  if a < 30 goto (15)
(9)  goto (18)
(12) if d > 20 goto (7)
(14) if d < 5 goto (17)
(16) if f < z*z goto (13)
```

所以入口语句为：
```
(1), (5), (7), (8), (10), (13), (15), (17), (18)
```

其中：
- `(1)` 是程序第一条；
- `(10), (15), (18), (7), (13), (17)` 是跳转目标；
- `(5), (8), (10), (13), (15), (17)` 是跳转语句的下一条。

去重并排序：
```
1, 5, 7, 8, 10, 13, 15, 17, 18
```

#### 划分基本块
按照“从一个入口语句开始，到下一个入口语句前一条结束”的规则，得到：
```
B1 = { (1), (2), (3), (4) }
B2 = { (5), (6) }
B3 = { (7) }
B4 = { (8), (9) }
B5 = { (10), (11), (12) }

B9 = { (18) }
```

### 什么是公共子表达式
如果表达式E在某次出现之前必然已经被计算过，并且E的分量在该次计算之后一致没有被改变，那么E的本次出现就是公共子表达式。如果上一次E的值赋给了x，且 x 的值至今没有被修改过，那么可以直接使用 x，而不必重新计算 E

### 死代码删除
死代码删除的关键是看一个赋值结果之后是否还可能被使用。若某变量赋值后再也不用，则该赋值是死代码。课件中也把“变量之后是否被使用”作为活跃性与死代码判断的核心依据。

### 循环优化
循环是优化的重要目标，因为循环体可能执行很多次，即使每轮只省一条指令，总收益也可能很多

![[Pasted image 20260621110444.png]]
```
LOAD R1, i
ADD  R1, R1, #1
MULT R1, R1, #4
LOAD R2, a(R1)
STORE x, R2
```
其中 `a(R1)` 表示地址 `base(a) + R1`，此时 `R1 = (i+1) * 4`，所以 `a(R1)` 就是 `a[i+1]`。

USE[B] = 在 B 中先被使用、后被定义的变量
DEF[B] = 在 B 中被赋值定义的变量

**B1**
```
d1: i = -1
d2: j = a
d3: a = u1

USE_B1 = {a, u1}
DEF_B1 = {i, j, a}
```

**B2**
```
d4: i = i + 1
d5: j = j - 1

USE_B2 = {i, j}
DEF_B2 = {i, j}
```

**B3**
```
d6: a = u2

USE_B3 = {u2}
DEF_B3 = {a}
```

**B4**
```
d7: i = u3

USE_B4 = {u3}
DEF_B4 = {i}
```

#### 活跃变量分析公式
活跃变量分析是**反向数据流分析**，从出口方向往入口方向传播。

```
公式：
OUT[B] = ∪ IN[S]          其中 S 是 B 的后继基本块
IN[B]  = USE[B] ∪ (OUT[B] - DEF[B])

边界条件
IN[EXIT] = ∅
```
