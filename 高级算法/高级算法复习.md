# 一、算法的最坏复杂度、平均复杂度、最优性

## 1. 最坏复杂度

最坏复杂度就是：在所有规模为 `n` 的输入中，算法执行时间最长的那种情况。

形式化地说：

[  
T_{worst}(n)=\max_{|I|=n}T(I)  
]

比如在一个长度为 `n` 的数组中顺序查找元素 `x`：

```text
for i = 1 to n:
    if A[i] == x:
        return i
return not found
```

如果 `x` 在最后一个位置，或者根本不在数组中，就需要比较 `n` 次。因此最坏复杂度是：

[  
O(n)  
]

最坏复杂度的意义是：它给你一个保证。无论输入多糟糕，算法最多也就消耗这么多时间。

---

## 2. 平均复杂度

平均复杂度不是简单地把最好和最坏平均一下，而是要考虑每种输入出现的概率。

[  
T_{avg}(n)=\sum_I P(I)T(I)  
]

老师在听写文本里强调：实际情况下输入概率通常很难知道，所以考试里一般会假设“均匀分布”。比如顺序查找时，假设目标元素等概率出现在第 `1,2,...,n` 个位置，那么平均比较次数为：

[  
\frac{1+2+\cdots+n}{n}=\frac{n+1}{2}  
]

所以平均复杂度仍然是：

[  
O(n)  
]

注意：平均复杂度必须说明概率模型。没有概率分布，平均复杂度就不严谨。

---

## 3. 算法最优性

所谓算法最优，不是说这个算法“看起来很好”，而是要证明两件事：

第一，所有能解决这个问题的算法，至少都需要这么多操作。

第二，你当前的算法正好达到了这个下界。

也就是：

[  
\text{算法上界}=\text{问题下界}  
]

那么这个算法就是最优的。

---

## 4. 例子：在 `n` 个数中找最大值

算法很简单：

```text
max = A[1]
for i = 2 to n:
    if A[i] > max:
        max = A[i]
return max
```

比较次数是：

[  
n-1  
]

为什么这是最优的？

因为要确定某个元素不是最大值，它至少要在一次比较中输给别人。除了真正的最大值以外，其余 `n-1` 个元素都必须被排除掉。因此至少需要 `n-1` 份“失败信息”。

一次比较最多只能产生一个失败者，所以至少需要：

[  
n-1  
]

次比较。

当前算法正好也是 `n-1` 次，所以它是最优的。

这就是老师强调的“用信息论思想证明最优性”。

# 二、渐进符号：O、Ω、Θ

复习提纲第二点中符号显示有缺失，但结合 PPT 和听写内容，显然是考：

[  
O,\ \Omega,\ \Theta  
]

以及相关性质证明。

---

## 1. 大 O：上界

[  
f(n)=O(g(n))  
]

表示 `f` 的增长速度不超过 `g` 的常数倍。

严格定义是：存在常数 `c>0` 和 `n_0`，使得当 `n≥n_0` 时：

[  
0\le f(n)\le c g(n)  
]

例如：

[  
3n^2+5n+7=O(n^2)  
]

证明：

当 `n≥1` 时：

[  
3n^2+5n+7\le 3n^2+5n^2+7n^2=15n^2  
]

所以取：

[  
c=15,\ n_0=1  
]

即可。

---

## 2. 大 Ω：下界

[  
f(n)=\Omega(g(n))  
]

表示 `f` 的增长速度至少达到 `g` 的常数倍。

也就是存在 `c>0` 和 `n_0`，使得当 `n≥n_0` 时：

[  
f(n)\ge c g(n)  
]

例如：

[  
3n^2+5n+7=\Omega(n^2)  
]

因为：

[  
3n^2+5n+7\ge 3n^2  
]

所以取：

[  
c=3,\ n_0=1  
]

即可。

---

## 3. Θ：紧确界

[  
f(n)=\Theta(g(n))  
]

表示 `f` 和 `g` 同阶。

也就是说：

[  
f(n)=O(g(n))  
]

并且：

[  
f(n)=\Omega(g(n))  
]

例如：

[  
3n^2+5n+7=\Theta(n^2)  
]

因为它既是 `O(n²)`，也是 `Ω(n²)`。

---

## 4. 常见性质

老师复习课里强调，考试可能更偏向考性质，而不是单纯背定义。

### 传递性

如果：

[  
f=O(g),\quad g=O(h)  
]

那么：

[  
f=O(h)  
]

同理，`Ω` 和 `Θ` 也有传递性。

---

### 对偶性

[  
f=O(g) \Longleftrightarrow g=\Omega(f)  
]

意思是：如果 `f` 不超过 `g`，那反过来 `g` 至少达到 `f`。

---

### Θ 的等价性

[  
f=\Theta(g)  
]

说明二者增长速度同阶，可以相互替换做渐进分析。

---

### 常数可以忽略

[  
cf(n)=\Theta(f(n))  
]

其中 `c` 是正常数。

例如：

[  
100n^2=\Theta(n^2)  
]

---

### 加法取最大阶

[  
f(n)+g(n)=\Theta(\max(f(n),g(n)))  
]

例如：

[  
n^2+n\log n+n=\Theta(n^2)  
]

因为最高阶是 `n²`。

---

## 5. 常见增长速度排序

从慢到快大致是：

[  
1<\log n<n^\alpha<n<n\log n<n^2<n^3<2^n<n!  
]

其中，`α>0`。

要记住：

- 对数函数增长最慢；
    
- 多项式函数比对数快；
    
- 指数函数比多项式快；
    
- 阶乘函数比指数还快。
    

---

# 三、递归关系求解：形似 Fibonacci 的递推

复习提纲第三点强调的是“形似斐波那契数列”的递归关系。老师听写里说得更具体：主要考**二阶线性齐次递推关系**，且一般只考两个不同根的情况。

---

## 1. 一般形式

常见形式：

[  
a_n=r_1a_{n-1}+r_2a_{n-2}  
]

例如 Fibonacci：

[  
F_n=F_{n-1}+F_{n-2}  
]

---

## 2. 特征方程法

对于：

[  
a_n=r_1a_{n-1}+r_2a_{n-2}  
]

假设解形如：

[  
a_n=x^n  
]

代入：

[  
x^n=r_1x^{n-1}+r_2x^{n-2}  
]

两边除以：

[  
x^{n-2}  
]

得到：

[  
x^2=r_1x+r_2  
]

整理为：

[  
x^2-r_1x-r_2=0  
]

这就是特征方程。

---

## 3. 两个不同根的情况

如果特征方程有两个不同根：

[  
s_1,\ s_2  
]

那么通解是：

[  
a_n=u s_1^n+v s_2^n  
]

其中 `u` 和 `v` 由初始条件决定。

---

## 4. Fibonacci 例子

Fibonacci：

[  
F_n=F_{n-1}+F_{n-2}  
]

特征方程：

[  
x^2=x+1  
]

即：

[  
x^2-x-1=0  
]

两个根为：

[  
x_1=\frac{1+\sqrt5}{2}  
]

[  
x_2=\frac{1-\sqrt5}{2}  
]

所以：

[  
F_n=u\left(\frac{1+\sqrt5}{2}\right)^n+  
v\left(\frac{1-\sqrt5}{2}\right)^n  
]

再由初始条件求出 `u,v`。

考试时通常不要求把最终常数算得特别复杂，但你要会写出特征方程、求根、写通解。

---

# 四、平滑函数与主定理

这个点来自递归与分治。老师强调两件事：

第一，会判断平滑函数。

第二，会把自然语言描述翻译成递归式，然后套主定理。

---

## 1. 平滑函数

如果函数 `f(n)` 满足：

[  
f(2n)=\Theta(f(n))  
]

则称 `f(n)` 是平滑函数。

直观理解：当输入规模扩大常数倍时，函数值也只是扩大常数倍，不会突然爆炸。

常见平滑函数：

[  
\log n,\ n,\ n\log n,\ n^\alpha  
]

其中：

[  
\alpha\ge0  
]

指数函数不是平滑函数。例如：

[  
f(n)=2^n  
]

则：

[  
f(2n)=2^{2n}=4^n  
]

它和 `2^n` 不是同阶。

---

## 2. 主定理的一般形式

分治递归常见形式：

[  
T(n)=aT(n/b)+f(n)  
]

其中：

- `a`：子问题个数；
    
- `n/b`：每个子问题规模；
    
- `f(n)`：划分、合并、非递归部分的代价。
    

关键是比较：

[  
f(n)  
]

和：

[  
n^{\log_b a}  
]

设：

[  
E=\log_b a  
]

那么比较的是：

[  
f(n)  
\quad \text{和} \quad  
n^E  
]

---

## 3. 三种情况

### 情况一：叶子层主导

如果：

[  
f(n)=O(n^{E-\epsilon})  
]

那么：

[  
T(n)=\Theta(n^E)  
]

意思是：每一层工作量往下越来越大，最后叶子层占主导。

例子：

[  
T(n)=8T(n/2)+n^2  
]

这里：

[  
a=8,\quad b=2  
]

[  
E=\log_2 8=3  
]

[  
f(n)=n^2  
]

因为：

[  
n^2=O(n^{3-1})  
]

所以：

[  
T(n)=\Theta(n^3)  
]

---

### 情况二：每层同阶

如果：

[  
f(n)=\Theta(n^E)  
]

那么：

[  
T(n)=\Theta(n^E\log n)  
]

例子：

[  
T(n)=2T(n/2)+n  
]

这里：

[  
E=\log_2 2=1  
]

[  
f(n)=n  
]

所以：

[  
T(n)=\Theta(n\log n)  
]

归并排序就是这个形式。

---

### 情况三：根层主导

如果：

[  
f(n)=\Omega(n^{E+\epsilon})  
]

并且满足正则条件：

[  
a f(n/b)\le c f(n),\quad c<1  
]

那么：

[  
T(n)=\Theta(f(n))  
]

例子：

[  
T(n)=2T(n/2)+n^2  
]

这里：

[  
E=1  
]

而：

[  
f(n)=n^2  
]

比 `n` 大一个多项式级别，所以：

[  
T(n)=\Theta(n^2)  
]

---

## 4. 老师可能怎么考

老师听写里提到一种考法：给你自然语言描述，让你翻译成递归式。

例如：

> 一个规模为 `n` 的问题，被划分成 4 个规模为 `n/3` 的子问题，合并代价是 `n²`。

你要写出：

[  
T(n)=4T(n/3)+n^2  
]

然后：

[  
a=4,\quad b=3,\quad f(n)=n^2  
]

[  
E=\log_3 4  
]

因为：

[  
n^2  
]

大于：

[  
n^{\log_3 4}  
]

所以属于主定理第三种情况：

[  
T(n)=\Theta(n^2)  
]

---

# 五、堆：定义、数组表示、堆操作

复习提纲第五点明确要求：了解堆的定义、如何用数组表示堆、如何实现堆上的操作 。

---

## 1. 堆的两个性质

堆是一种特殊的二叉树，通常指最大堆或最小堆。

以最大堆为例，它满足两个性质。

---

### 结构性质

堆是一棵接近完全的二叉树。

更具体地说：

- 除最后一层外，其余层都是满的；
    
- 最后一层节点尽量靠左；
    
- 所有叶子节点只可能出现在最后一层或倒数第二层。
    

---

### 堆序性质

最大堆要求：

[  
parent \ge child  
]

也就是父节点的值大于等于子节点的值。

因此最大堆的根节点一定是整个堆中的最大元素。

最小堆则相反：

[  
parent \le child  
]

---

## 2. 数组表示堆

由于堆是完全二叉树，所以可以按层序存储到数组中，不需要显式存左右指针。

如果数组下标从 `1` 开始：

[  
parent(i)=\lfloor i/2 \rfloor  
]

[  
left(i)=2i  
]

[  
right(i)=2i+1  
]

如果数组下标从 `0` 开始：

[  
parent(i)=\lfloor (i-1)/2 \rfloor  
]

[  
left(i)=2i+1  
]

[  
right(i)=2i+2  
]

老师听写里特别强调：用数组模拟堆，本质是用数组下标关系模拟树上的父子关系。

---

## 3. 删除最大值 deleteMax

最大堆删除最大值时，不能真的把根节点这个“结构位置”删掉，否则树结构会散掉。

正确做法是：

1. 取出根节点的值；
    
2. 用最后一个节点的值填到根节点；
    
3. 删除最后一个节点；
    
4. 从根开始向下调整，恢复堆序性质。
    

伪代码：

```text
deleteMax(H):
    max = H[1]
    H[1] = H[size]
    size--
    fixDown(1)
    return max
```

向下调整：

```text
fixDown(i):
    while left(i) <= size:
        child = left(i)
        if right(i) <= size and H[right(i)] > H[left(i)]:
            child = right(i)

        if H[i] >= H[child]:
            break

        swap(H[i], H[child])
        i = child
```

复杂度：

[  
O(\log n)  
]

---

## 4. 插入 insert

插入新元素时：

1. 把新元素放到数组末尾；
    
2. 从末尾向上调整。
    

```text
insert(x):
    size++
    H[size] = x
    i = size
    while i > 1 and H[parent(i)] < H[i]:
        swap(H[parent(i)], H[i])
        i = parent(i)
```

复杂度：

[  
O(\log n)  
]

---

## 5. 建堆 buildHeap

建堆不是从前往后逐个插入，而是从最后一个非叶子节点开始向下调整。

```text
buildHeap(H):
    for i = floor(size/2) downto 1:
        fixDown(i)
```

总复杂度是：

[  
O(n)  
]

不是：

[  
O(n\log n)  
]

原因是：大量节点在底层，高度很小；只有少数节点在上层，高度较大。把每个节点的下沉成本加起来，总和是线性的。

---

## 6. 堆排序

堆排序步骤：

1. 建最大堆；
    
2. 反复取出最大值；
    
3. 把最大值放到数组末尾；
    
4. 缩小堆范围；
    
5. 继续调整。
    

复杂度：

[  
O(n\log n)  
]

空间复杂度：

[  
O(1)  
]

特点：

- 最坏时间稳定；
    
- 原地排序；
    
- 不是稳定排序；
    
- 理论优秀，但实际常数通常比快排大。
    

---

# 六、复用快排平均复杂度分析

复习提纲第六点说的是：复用快速排序中平均复杂度分析的方法处理其他问题。

老师听写里提到类似“螺丝和螺母匹配”“试穿衣服匹配”的问题。这类问题的本质是：不能直接比较同类，只能比较异类，但可以用类似快排的随机划分思想。

---

## 1. 快排平均复杂度回顾

快排基本思想：

1. 选一个 pivot；
    
2. 小于 pivot 的放左边；
    
3. 大于 pivot 的放右边；
    
4. 对左右两边递归排序。
    

如果 pivot 左边有 `i` 个元素，右边有 `n-1-i` 个元素，则：

[  
T(n)=T(i)+T(n-1-i)+O(n)  
]

平均情况下，假设 pivot 排名均匀分布：

[  
E[T(n)]=\frac{1}{n}\sum_{i=0}^{n-1}(E[T(i)]+E[T(n-1-i)])+O(n)  
]

最终：

[  
E[T(n)]=O(n\log n)  
]

老师强调：推导过程不一定要全部会，但要会把新问题转化成快排式的递归。

---

## 2. 螺丝和螺母匹配问题

问题：

- 有 `n` 个螺丝和 `n` 个螺母；
    
- 每个螺丝恰好匹配一个螺母；
    
- 不能比较两个螺丝；
    
- 不能比较两个螺母；
    
- 只能比较一个螺丝和一个螺母。
    

目标：把所有螺丝和螺母配对。

---

## 3. 类快排算法

算法：

```text
match(screws, nuts):
    if size <= 1:
        return

    randomly choose a screw s

    use s to partition all nuts:
        smaller nuts
        matching nut m
        larger nuts

    use m to partition all screws:
        smaller screws
        matching screw s
        larger screws

    recursively match smaller parts
    recursively match larger parts
```

这里的关键是：

- 随机选一个螺丝，相当于快排选 pivot；
    
- 用这个螺丝找到匹配螺母；
    
- 再用匹配螺母划分螺丝；
    
- 左右两边递归。
    

平均复杂度与快排一样：

[  
O(n\log n)  
]

---

## 4. 试穿衣服问题

老师听写里举过类似例子：`n` 个同学和 `n` 套衣服，衣服标签乱了，每个人只能通过试穿判断衣服大小是否合适。

这和螺丝螺母问题等价：

- 人相当于螺丝；
    
- 衣服相当于螺母；
    
- 只能“人试衣服”，不能直接比较两个人或两件衣服。
    

也可以用快排式算法：

1. 随机选一个人；
    
2. 让他试穿所有衣服，找到合适衣服，并把衣服分成偏小、合适、偏大；
    
3. 用这件衣服去比较所有人，把人分成偏瘦、合适、偏胖；
    
4. 递归处理两边。
    

平均试穿次数：

[  
O(n\log n)  
]

# 七、对抗策略：最大最小、第二大

复习提纲第七点明确要求：解释对抗策略，并用它分析“同时求最大最小”和“求第二大元素” 。

---

## 1. 什么是对抗策略

对抗策略，也叫 adversary argument。

它常用于证明问题下界。

基本思想：

算法可以看成一个不断获取信息的过程。算法想尽快获得足够信息，对手则尽量让算法每一步获得的信息最少，从而迫使算法做更多操作。

也就是说：

> 当算法面对多种可能结果时，对手总是选择让算法最不舒服、获得信息最少的那种结果。

---

## 2. 同时求最大值和最小值

### 朴素算法

先找最大值，再找最小值：

[  
(n-1)+(n-1)=2n-2  
]

次比较。

---

### 最优算法：成对比较

假设 `n` 是偶数。

第一步，把元素两两分组，每组内部比较一次。

每组比较后：

- 较大者进入“最大值候选组”；
    
- 较小者进入“最小值候选组”。
    

第一阶段比较：

[  
n/2  
]

次。

然后在 `n/2` 个较大者中找最大值，需要：

[  
n/2-1  
]

次。

在 `n/2` 个较小者中找最小值，需要：

[  
n/2-1  
]

次。

总比较次数：

[  
n/2+(n/2-1)+(n/2-1)=\frac{3n}{2}-2  
]

---

### 下界证明

要确定最大值，除最大值以外的 `n-1` 个元素都必须输过一次。

所以需要：

[  
n-1  
]

份“输的信息”。

要确定最小值，除最小值以外的 `n-1` 个元素都必须赢过一次。

所以需要：

[  
n-1  
]

份“赢的信息”。

总共需要：

[  
2n-2  
]

份信息。

但是一次比较最多能提供两份有用信息：一个赢，一个输。

对抗策略会让你尽量少获得双份信息。只有两个完全没有比较过的“空白元素”第一次比较时，才能同时有效产生一份赢信息和一份输信息。这样的比较最多有：

[  
n/2  
]

次。

之后每次比较，在对抗策略下最多只能新增一份有效信息。

所以至少需要：

[  
\frac n2 + (2n-2-n)=\frac{3n}{2}-2  
]

次比较。

因此成对比较算法是最优的。

---

## 3. 求第二大元素

### 朴素方法

先找最大值，再在剩下的 `n-1` 个元素中找最大值：

[  
(n-1)+(n-2)=2n-3  
]

次比较。

这不是最优。

---

### 最优方法：锦标赛法

把所有元素组织成淘汰赛。

最大值一路获胜。

求最大值需要：

[  
n-1  
]

次比较。

第二大元素一定只可能是**直接输给最大值**的那些元素。

如果 `n` 是 2 的幂，最大值一共打了：

[  
\log_2 n  
]

场比赛。

所以第二大只可能在这 `log n` 个元素中产生。

再从这些候选中找最大：

[  
\log_2 n-1  
]

次比较。

总比较次数：

[  
n+\log_2 n-2  
]

一般写作：

[  
n+\lceil \log_2 n\rceil-2  
]

---

### 下界证明思路

任何算法要找第二大，必须先能确定最大。

确定最大至少要：

[  
n-1  
]

次比较。

而第二大元素必须直接输给最大值。否则如果它输给了其他元素，就无法保证它是第二大。

对抗策略可以迫使最大值至少直接比较：

[  
\lceil \log_2 n\rceil  
]

个不同元素。

因此还要在这些输给最大值的候选中找最大，需要：

[  
\lceil \log_2 n\rceil-1  
]

次比较。

所以总下界是：

[  
n+\lceil \log_2 n\rceil-2  
]

锦标赛算法达到这个下界，因此是最优的。

---

# 八、线性时间选择算法

复习提纲第八点是线性时间选择算法的实现和应用 。

---

## 1. 选择问题

给定数组 `A` 和整数 `k`，找第 `k` 小的元素。

例如：

```text
A = [7, 2, 9, 4, 5]
k = 3
```

第 3 小是 `5`。

特殊情况：

- `k=1`：找最小值；
    
- `k=n`：找最大值；
    
- `k=ceil(n/2)`：找中位数。
    

---

## 2. QuickSelect

随机选择算法类似快排。

```text
select(A, k):
    choose pivot p
    partition A into L, E, G

    if k <= |L|:
        return select(L, k)
    else if k <= |L| + |E|:
        return p
    else:
        return select(G, k - |L| - |E|)
```

平均复杂度：

[  
O(n)  
]

最坏复杂度：

[  
O(n^2)  
]

因为每次只递归一边，不像快排那样两边都递归。

---

## 3. 确定性线性选择算法：Median of Medians

老师听写中说得很明确：把元素按 5 个一组分组，每组找中位数，再找这些中位数的中位数，用它做划分元素。

步骤如下：

1. 把 `n` 个元素每 5 个分一组；
    
2. 每组排序，找出组内中位数；
    
3. 得到大约 `n/5` 个中位数；
    
4. 递归找这些中位数的中位数 `m`；
    
5. 用 `m` 作为 pivot 划分原数组；
    
6. 只递归进入包含第 `k` 小元素的一边。
    

伪代码：

```text
select(A, k):
    if |A| is small:
        sort A
        return A[k]

    divide A into groups of 5
    find median of each group
    m = select(medians, ceil(len(medians)/2))

    partition A into L, E, G around m

    if k <= |L|:
        return select(L, k)
    else if k <= |L| + |E|:
        return m
    else:
        return select(G, k - |L| - |E|)
```

---

## 4. 为什么是线性时间

关键结论：

用“中位数的中位数”作为 pivot，可以保证划分不会太差。

至少有大约：

[  
3n/10  
]

个元素不大于 pivot，也至少有大约：

[  
3n/10  
]

个元素不小于 pivot。

因此递归进入的最大子问题规模不超过：

[  
7n/10  
]

于是递归式为：

[  
T(n)\le T(n/5)+T(7n/10)+O(n)  
]

这个递归式的解是：

[  
T(n)=O(n)  
]

---

## 5. 典型应用

如果题目说：

> 给定 `n` 个互不相同的数，在线性时间内找出最大的 20 个。

做法：

1. 找第 `n-20+1` 小的元素，也就是第 `n-19` 小；
    
2. 这个元素作为阈值；
    
3. 扫描一遍数组，输出所有大于等于它的元素。
    

选择阈值：

[  
O(n)  
]

扫描：

[  
O(n)  
]

总复杂度：

[  
O(n)  
]

如果是“删除较大的一半”，也可以先找中位数，再线性扫描删除大于中位数的元素。

# 九、平摊分析：MultiPop、DeleteLargerHalf、双栈队列

复习提纲第九点明确考平摊分析，并列出三个经典问题：MultiPop Stack、DeleteLargerHalf、栈模拟队列 。听写文本中老师也强调这一点是三选一 。

---

## 1. 平摊分析的含义

平摊分析研究的是一串操作的总成本，而不是某一次操作的最坏成本。

它不依赖概率，所以不同于平均复杂度。

如果 `m` 次操作总成本是：

[  
O(m)  
]

那么每次操作的平摊复杂度就是：

[  
O(1)  
]

直观理解：某些操作偶尔很贵，但贵操作不能频繁发生，它的代价可以摊到之前或之后的便宜操作上。

---

## 2. MultiPop Stack

栈支持三种操作：

```text
Push(x)
Pop()
MultiPop(k)
```

`MultiPop(k)` 表示一次弹出最多 `k` 个元素。

单次 `MultiPop(k)` 最坏可能是：

[  
O(n)  
]

因为可能一次弹出很多元素。

但是对一整个操作序列来说，每个元素：

- 最多被 Push 一次；
    
- 最多被 Pop 或 MultiPop 弹出一次。
    

所以每个元素最多承担常数次操作。

如果有 `m` 次操作，总成本是：

[  
O(m)  
]

平摊每次：

[  
O(1)  
]

---

## 3. DeleteLargerHalf

问题通常是：

维护一个集合，支持插入元素和删除较大的一半元素。

一次 DeleteLargerHalf 可能删除很多元素，看起来最坏是：

[  
O(n)  
]

但是每个元素一旦被删除，就不会再被删除第二次。

如果整个过程中插入了 `n` 个元素，那么所有删除操作删掉的元素总数最多是 `n`。

所以总删除成本：

[  
O(n)  
]

平摊到每次插入 / 删除操作上，通常是：

[  
O(1)  
]

当然，如果底层数据结构需要额外维护排序或选择，中间可能还有别的成本；但这道复习题关注的是“删除很多元素的总成本可以摊掉”。

---

## 4. 两个栈模拟队列

用两个栈实现队列：

```text
stack_in
stack_out
```

入队：

```text
enqueue(x):
    stack_in.push(x)
```

出队：

```text
dequeue():
    if stack_out is empty:
        while stack_in is not empty:
            stack_out.push(stack_in.pop())
    return stack_out.pop()
```

单次 `dequeue` 最坏可能是：

[  
O(n)  
]

因为当 `stack_out` 为空时，要把 `stack_in` 中所有元素倒过去。

但每个元素：

1. 进入 `stack_in` 一次；
    
2. 从 `stack_in` 弹出一次；
    
3. 进入 `stack_out` 一次；
    
4. 从 `stack_out` 弹出一次。
    

每个元素总共只移动常数次。

所以 `m` 次队列操作总成本：

[  
O(m)  
]

平摊每次：

[  
O(1)  
]

---

# 十、图搜索：DFS、BFS、拓扑序、任务调度、关键路径、强连通区域

复习提纲第十点要求掌握 DFS/BFS 思想，并能解决 topo 序、任务调度、关键路径、强连通区域问题 。老师听写文本强调：无向图关节点不考，重点是有向图上的四类问题 。

---

## 1. DFS：深度优先搜索

DFS 的思想是：沿着一条路径一直往深处走，走不动了再回溯。

伪代码：

```text
DFS(v):
    mark v as discovered

    for each neighbor w of v:
        if w is undiscovered:
            DFS(w)

    mark v as finished
```

DFS 中顶点通常有三种状态：

- undiscovered：未发现；
    
- discovered：已发现但未完成；
    
- finished：已完成。
    

复杂度：

[  
O(V+E)  
]

如果用邻接矩阵，则可能是：

[  
O(V^2)  
]

---

## 2. BFS：广度优先搜索

BFS 的思想是：从起点开始，一层一层向外扩展。

使用队列。

```text
BFS(s):
    mark s as discovered
    enqueue(s)

    while queue not empty:
        v = dequeue()
        for each neighbor w of v:
            if w is undiscovered:
                mark w as discovered
                enqueue(w)
        mark v as finished
```

复杂度：

[  
O(V+E)  
]

BFS 常用于无权图最短路径，因为它第一次访问到某个点时，路径边数最少。

---

## 3. 拓扑排序 topo order

拓扑排序只适用于 DAG，即有向无环图。

定义：

若图中有边：

[  
u\to v  
]

则在拓扑序中，`u` 必须排在 `v` 前面。

---

### DFS 做拓扑排序

DFS 完成某个顶点时，把它压入栈。

```text
DFS(v):
    visited[v] = true
    for w in adj[v]:
        if not visited[w]:
            DFS(w)
    push v into stack
```

最后弹栈得到拓扑序。

本质原因：一个点完成时，它能到达的后继都已经完成了，所以完成时间逆序就是拓扑序。

复杂度：

[  
O(V+E)  
]

---

### 入度法

也可以用入度为 0 的点。

```text
topoSort(G):
    Q = all vertices with indegree 0

    while Q not empty:
        v = dequeue(Q)
        output v

        for each edge v -> w:
            indegree[w]--
            if indegree[w] == 0:
                enqueue(w)
```

如果输出的顶点数少于 `V`，说明图中有环。

---

## 4. 任务调度

任务调度可以转化为拓扑排序。

如果任务 `A` 必须在任务 `B` 之前完成，就建边：

[  
A\to B  
]

然后对任务图做拓扑排序。

若存在拓扑序，则说明任务可以被合法安排。

若图中有环，例如：

[  
A\to B,\ B\to C,\ C\to A  
]

则无法安排，因为任务之间互相依赖。

---

## 5. 关键路径

关键路径通常出现在带权 DAG 中。

一个项目由多个任务构成，边或点有持续时间。关键路径是决定整个项目总耗时的最长依赖路径。

如果关键路径上的任务延迟，整个项目会延迟。

如果非关键路径上的任务缩短，项目总时间不一定缩短。

---

### 基本算法

对 DAG 按拓扑序做动态规划。

如果任务持续时间在点上，设：

[  
time[v]  
]

表示任务 `v` 的耗时。

设：

[  
eft[v]  
]

表示任务 `v` 的最早完成时间。

则：

[  
eft[v]=time[v]+\max_{u\to v} eft[u]  
]

如果 `v` 没有前驱：

[  
eft[v]=time[v]  
]

同时记录使 `eft[v]` 最大的前驱：

[  
critDep[v]  
]

最后找到 `eft` 最大的顶点，从它沿着 `critDep` 反向追踪，就得到关键路径。

复杂度：

[  
O(V+E)  
]

---

## 6. 强连通分量 SCC

在有向图中，如果顶点集合内部任意两个点都互相可达，则这个集合是强连通分量。

也就是说，对于同一个 SCC 内的任意 `u,v`：

[  
u\leadsto v  
]

并且：

[  
v\leadsto u  
]

---

### Kosaraju 算法

步骤：

1. 对原图 `G` 做 DFS；
    
2. 按完成时间把顶点压栈；
    
3. 构造反图 `G^T`；
    
4. 按栈中顺序，在反图上 DFS；
    
5. 每次 DFS 得到一个强连通分量。
    

伪代码：

```text
strongComponents(G):
    stack = empty

    DFS on G:
        when v is finished:
            push v into stack

    GT = transpose(G)

    clear visited

    while stack not empty:
        v = pop(stack)
        if not visited[v]:
            run DFS(GT, v)
            all reached vertices form one SCC
```

复杂度：

[  
O(V+E)  
]

# 十一、贪心法：换硬币、单源最短路径

复习提纲第十一点要求掌握贪心思想和特点，并用贪心解决换硬币、单源最短路径 。听写文本中老师进一步强调贪心聚焦这两个问题 。

---

## 1. 贪心算法思想

贪心算法每一步都选择当前看起来最好的选项，并且选择后不再反悔。

它的核心特点是：

- 局部最优；
    
- 逐步构造解；
    
- 选择不可撤销；
    
- 希望局部最优能推出全局最优。
    

但贪心不是万能的。必须证明它正确。

常见证明方法是交换论证：

假设存在一个最优解没有采用贪心选择。然后证明可以把它改造成包含贪心选择的另一个最优解，而且不会变差。

---

## 2. 换硬币问题

硬币面值为：

[  
1,5,10,25  
]

每种硬币数量无限。

目标：用最少硬币数凑出金额 `n`。

贪心策略：

> 每次选择不超过剩余金额的最大面值硬币。

例如：

[  
96=25+25+25+10+10+1  
]

共 6 枚硬币。

---

## 3. 贪心并不总是正确

如果硬币系统是：

[  
1,5,12  
]

金额是：

[  
15  
]

贪心会选：

[  
12+1+1+1  
]

共 4 枚。

但最优解是：

[  
5+5+5  
]

共 3 枚。

所以考试中要注意：不是所有换硬币都能用贪心。只有特定币制可以。

---

## 4. 换硬币的证明思路

对于 `{1,5,10,25}`，证明思路通常是：

1. 最优解中 1 分硬币不会超过 4 个，否则 5 个 1 可以换成 1 个 5；
    
2. 5 分硬币不会超过 1 个，否则 2 个 5 可以换成 1 个 10；
    
3. 10 分硬币不会超过 2 个，否则某些组合可以换成 25 或更少硬币；
    
4. 因此对于较大金额，尽量使用 25 分硬币不会让硬币数变差；
    
5. 对剩余金额递归应用同样策略。
    

考试不一定要求非常严密的完整证明，但要写出“交换替换”的思想。

---

## 5. 单源最短路径：Dijkstra 算法

问题：

给定带权有向图 `G=(V,E)` 和源点 `s`，求从 `s` 到所有顶点的最短路径。

前提：

[  
w(e)\ge0  
]

所有边权非负。

---

### 算法思想

维护两个集合：

- 已确定最短距离的点集合 `S`；
    
- 未确定的点集合 `V-S`。
    

每次选择当前 `dist` 最小的未确定点 `u`，把它加入 `S`，然后用它松弛邻接边。

```text
Dijkstra(G, s):
    for each vertex v:
        dist[v] = infinity
    dist[s] = 0

    S = empty

    while S != V:
        u = vertex outside S with minimum dist[u]
        add u to S

        for each edge u -> v:
            if dist[v] > dist[u] + w(u,v):
                dist[v] = dist[u] + w(u,v)
                parent[v] = u
```

---

### 正确性证明思路

为什么每次选出的 `u` 可以被永久确定？

假设 `u` 当前是所有未确定点中 `dist` 最小的，但它其实还有一条更短路径。

那条更短路径从源点出发，一定会第一次跨出已确定集合 `S`，到达某个未确定点 `x`。

由于所有边权非负，所以：

[  
dist[x]\le \text{那条路径到 }u\text{ 的长度}  
]

而 `u` 是未确定点中 `dist` 最小的，所以：

[  
dist[u]\le dist[x]  
]

矛盾。

因此 `u` 的最短距离可以确定。

---

### 复杂度

如果用数组找最小 `dist`：

[  
O(V^2)  
]

如果用堆优化：

[  
O((V+E)\log V)  
]

考试如果给图让你跑 Dijkstra，一般要能列出每一步的 `dist` 表和已确定集合。

# 十二、动态规划：LCS、抽牌、回文

## 1. 动态规划的思想

动态规划适合解决具有以下特征的问题：

1. 最优子结构；
    
2. 重叠子问题；
    
3. 子问题之间存在依赖顺序；
    
4. 可以把递归转化为填表。
    

动态规划答题模板：

```text
第一步：定义状态
第二步：写递推关系
第三步：说明初始化和计算顺序
第四步：给出答案位置
第五步：分析复杂度
```

## 2. 最长公共子序列 LCS

给定两个序列：

[  
X=x_1x_2...x_m  
]

[  
Y=y_1y_2...y_n  
]

求它们的最长公共子序列长度。

注意：子序列不要求连续，但相对顺序不能变。

---

### 状态定义

[  
dp[i][j]  
]

表示：

[  
X[1..i]  
]

和：

[  
Y[1..j]  
]

的最长公共子序列长度。

---

### 递推式

如果：

[  
x_i=y_j  
]

那么：

[  
dp[i][j]=dp[i-1][j-1]+1  
]

如果：

[  
x_i\ne y_j  
]

那么：

[  
dp[i][j]=\max(dp[i-1][j],dp[i][j-1])  
]

---

### 初始化

[  
dp[0][j]=0  
]

[  
dp[i][0]=0  
]

因为空序列和任何序列的 LCS 长度都是 0。

---

### 计算顺序

按 `i` 从小到大、`j` 从小到大填表。

```text
for i = 1 to m:
    for j = 1 to n:
        if X[i] == Y[j]:
            dp[i][j] = dp[i-1][j-1] + 1
        else:
            dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

答案：

[  
dp[m][n]  
]

复杂度：

[  
O(mn)  
]

---

## 3. 抽牌拿取最大值问题

问题：

有一排牌：

[  
a_1,a_2,...,a_n  
]

两个人轮流拿牌，每次只能从最左端或最右端拿一张，双方都采用最优策略。问先手最多能拿多少，或者先手是否能赢。

---

### 方法一：差值 DP

定义：

[  
dp[i][j]  
]

表示当前玩家面对区间：

[  
[i,j]  
]

时，最多能比对手多拿多少分。

如果当前玩家拿左边：

[  
a_i  
]

那么对手面对：

[  
[i+1,j]  
]

对手相对于当前玩家能多拿：

[  
dp[i+1][j]  
]

所以当前玩家净优势是：

[  
a_i-dp[i+1][j]  
]

如果当前玩家拿右边：

[  
a_j  
]

净优势是：

[  
a_j-dp[i][j-1]  
]

因此：

[  
dp[i][j]=\max(a_i-dp[i+1][j],\ a_j-dp[i][j-1])  
]

---

### 初始化

[  
dp[i][i]=a_i  
]

只有一张牌时，当前玩家直接拿走。

---

### 计算顺序

按区间长度从小到大：

```text
for i = 1 to n:
    dp[i][i] = a[i]

for len = 2 to n:
    for i = 1 to n-len+1:
        j = i + len - 1
        dp[i][j] = max(a[i] - dp[i+1][j],
                       a[j] - dp[i][j-1])
```

如果：

[  
dp[1][n]\ge0  
]

则先手不输。

---

### 方法二：直接求先手最大得分

也可以定义：

[  
F[i][j]  
]

表示当前玩家在区间 `[i,j]` 中最多能拿到的总分。

当玩家拿左边后，对手也会最优，所以当前玩家后续只能拿到两个子局面中的较小结果：

[  
F[i][j]=\max(  
a_i+\min(F[i+2][j],F[i+1][j-1]),  
a_j+\min(F[i+1][j-1],F[i][j-2])  
)  
]

但这个式子更复杂，考试中更推荐使用差值 DP。

复杂度：

[  
O(n^2)  
]

## 4. 回文判断

给定字符串：

[  
s[1..n]  
]

判断任意子串：

[  
s[i..j]  
]

是否为回文。

---

### 状态定义

[  
pal[i][j]=true  
]

表示：

[  
s[i..j]  
]

是回文。

---

### 递推式

如果：

[  
s[i]\ne s[j]  
]

则：

[  
pal[i][j]=false  
]

如果：

[  
s[i]=s[j]  
]

则需要继续看内部：

[  
pal[i][j]=pal[i+1][j-1]  
]

所以：

[  
pal[i][j]=(s[i]=s[j])\land pal[i+1][j-1]  
]

---

### 初始化

单个字符一定是回文：

[  
pal[i][i]=true  
]

空串可以视为回文：

[  
pal[i][i-1]=true  
]

长度为 2 时：

[  
pal[i][i+1]=(s[i]=s[i+1])  
]

---

### 计算顺序

因为：

[  
pal[i][j]  
]

依赖：

[  
pal[i+1][j-1]  
]

所以应该按子串长度从小到大计算。

```text
for i = 1 to n:
    pal[i][i] = true

for len = 2 to n:
    for i = 1 to n-len+1:
        j = i + len - 1
        pal[i][j] = (s[i] == s[j]) and (len == 2 or pal[i+1][j-1])
```

复杂度：

[  
O(n^2)  
]

## 5. 回文最少分割

问题：

给定字符串 `s`，把它切成若干段，使每一段都是回文，求最少切几刀。

例如：

```text
aab
```

可以切成：

```text
aa | b
```

只需要 1 刀。

---

### 第一步：预处理回文表

先用刚才的方法求出：

[  
pal[i][j]  
]

表示：

[  
s[i..j]  
]

是否是回文。

---

### 第二步：定义最少切割状态

一种写法是：

[  
dp[i]  
]

表示前缀：

[  
s[1..i]  
]

最少需要切几刀。

如果：

[  
pal[1][i]=true  
]

说明整个前缀本身就是回文：

[  
dp[i]=0  
]

否则：

[  
dp[i]=\min_{1\le j<i,\ pal[j+1][i]=true}(dp[j]+1)  
]

意思是：最后一刀切在 `j` 后面，要求最后一段 `s[j+1..i]` 是回文。

---

### 伪代码

```text
compute pal[i][j]

for i = 1 to n:
    if pal[1][i]:
        dp[i] = 0
    else:
        dp[i] = infinity
        for j = 1 to i-1:
            if pal[j+1][i]:
                dp[i] = min(dp[i], dp[j] + 1)
```

答案：

[  
dp[n]  
]

复杂度：

[  
O(n^2)  
]
