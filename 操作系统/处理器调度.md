[[处理器管理]]
当系统中有多个进程或线程都想使用CPU时，操作系统应该让谁运行、运行多久、什么时候换人？

## 什么是处理器调度
**处理器调度**就是操作系统按照某种策略，把CPU分配给某个可运行进程或线程的过程
在单核CPU上，同一时刻只能有一个进程或线程真正占用CPU，可是系统中通常同时存在很多进程

这些进程看起来“同时运行”，实际上是操作系统通过快速切换CPU使用权，让它们交替执行
所以处理器调度要解决的问题是：

```
谁可以进入系统？
谁可以留在内存？
谁现在获得CPU？
谁应该被抢占？
一个进程应该运行多久？
```

## 处理器调度的层次

```
高级调度 / 长程调度 / 作业调度
中级调度 / 中程调度 / 平衡负载调度
低级调度 / 短程调度 / 进程调度
```

### 高级调度：决定能不能进入系统
**高级调度**又叫 **长程调度** 或 **作业调度**

决定能否加入到执行的进程池中。批处理系统中，高级调度会从后备作业队列中选择作业进入主存，并为作业运行前做准备、运行后做善后工作
分时系统中，高级调度可能决定是否接受中断用户连接、命令能否被系统接纳并构成进程、新建态进程是否加入就绪队列

**批处理系统中的高级调度**
在批处理系统中，用户提交作业后，作业不一定立刻进入内存运行
可能有一个后备作业队列：
`后备作业队列:J1,J2,J3,J4,...`
高级调度负责从中选择一部分进入内存

选择时可能考虑：
- 作业优先级
- 预计运行时间
- 资源需求
- I/O 需求
- 系统负载
- 公平性

例如：系统可能不希望同时放入太多CPU密集型作业，否则I/O 设备利用率很低；也不希望同时放入太多I/O密集型作业；否则CPU可能空闲

**分时系统中的高级调度**
在分时系统中，高级调度表现为：
```
是否允许一个用户登录？
是否允许一个命令创建进程？
系统是否还有足够资源接纳新进程？
```

例如：系统负载过高时，可能暂时拒绝新的登录或限制新任务创建

**高级调度的特点**
高级调度频率较低
因为作业进入系统不是每毫秒都发生的事情，它更多影响系统整体负载和作业吞吐量

`高级调度：控制系统入口`

### 中级调度：决定“能不能留在内存”
**中级调度** 又叫 **中程调度** 或 **平衡负载调度**

核心问题：
```
哪些进程应该驻留在内存中？
哪些进程应该被换出道外存挂起？
```
中程调度决定主存中的可用进程集合；引入中级调度是为了提高内存利用率和作业吞吐量，它决定哪些进程被允许驻留主存参与竞争处理器及其他资源，并通过把部分进程换出主存进入挂起状态来平顺系统负载

**为什么需要中级调度**
假设系统中有很多进程都在主存中：
`P1,P2,P3,P4,...,P100`
如果内存不足，系统会频繁换页，导致大量时间花在磁盘的换入换出上，CPU反而没多少时间真正执行程序，这种现象严重时叫 **抖动**

中级调度的作用就是：
```
把一部风进程换出内存
降低内存压力
减少系统抖动
提高整体吞吐量
```

**中级调度和挂起状态**
中级调度和 **挂起状态** 密切相关
它可能导致：
```
就绪态 -> 挂起就绪态
等待态 -> 挂起等待态
挂起就绪态 -> 就绪态
挂起等待态 -> 等待态 或 挂起就绪态
```

一个等待 I/O 的进程暂时用不到 CPU，也许可以先换出内存
`等待态 -> 挂起等待态`
等它等待的事件完成后：
`挂起等待态 -> 挂起就绪态`
之后系统负载降低，再把它调回内存：
`挂起就绪态 -> 就绪态`

**中级调度的特点**
中级调度频率介于高级调度和低级调度之间
它不是每次CPU切换时都发生，但比高级调度更频繁

`中级调度：控制内存驻留集合`

### 低级调度：决定“现在谁用CPU”
**低级调度** 又叫 **短程调度、进程调度** 或 **处理器调度**

它的核心问题时：
`当前CPU应该分配给哪个就绪进程或内核级进程？`

低级调度按照某种原则把处理器分派给就绪态进程或内核级线程。进程调度程序又称分派程序，是操作系统中实现处理器调度的程序，是操作系统最核心部分
