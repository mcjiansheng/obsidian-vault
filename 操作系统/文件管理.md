[[操作系统]]
> **文件是操作系统提供给用户和程序的数据抽象；文件系统是操作系统中负责管理这些文件的模块。**

## 文件的概念
**为什么操作系统要引入“文件”？**
如果没有文件，用户要保存一段数据，就必须知道它放在磁盘的哪个柱面、哪个磁道、哪个扇区，甚至要自己考虑磁盘块如何分配、如何回收、如何防止别人误删。这显然不现实。

所以操作系统引入文件，把底层复杂的辅助存储器细节隐藏起来。

**文件系统的首要需求是按名存取**

### 文件的定义
文件是具有符号名的、在逻辑上具有完整意义的一组相关信息项的序列。

**具有符号名**
文件不是靠物理地址被用户识别，而是靠**文件名**识别。

文件名是用户和程序访问文件的入口。没有名字，用户就无法方便地引用文件。

**逻辑上具有完整意义**
文件不是任意堆放的一串二进制位，而是对用户或程序有意义的数据集合。

也就是说，文件是站在**用户视角和程序视角**看待数据的单位。

**一组相关信息项**
文件内部的数据项彼此相关。例如学生成绩文件中，每条记录可能包含学号、姓名、课程、成绩；源程序文件中，每一行代码都属于同一个程序；图片文件中，每部分二进制数据共同构成一张图像。

**序列**
文件中的信息通常具有某种顺序。比如文本文件按字节顺序排列，记录式文件按记录顺序排列，可执行文件按特定格式组织代码段、数据段、符号表等内容。

所以，文件既不是纯粹的“字节堆”，也不是纯粹的“磁盘块集合”，而是**用户逻辑意义和物理存储结构之间的桥梁**。

### 文件 document 与计算机文件 file 的区别
普通文件更偏向现实世界中的文档、档案、材料，例如纸质申请表、合同、论文。

计算机文件则是操作系统管理的数据对象。它可以是文本文档，也可以是程序、图片、音频、视频、数据库文件、设备文件，甚至在 Unix/Linux 中，很多设备也可以被抽象为文件。

### 文件的命名
文件名通常由**字母、数字和其他符号组成**，具体格式和长度因操作系统而异。文件命名一般包括**文件名和扩展名**
`文件名.扩展名`
#### 文件名的作用
文件名主要用于**识别文件**。
#### 扩展名的作用
扩展名通常用于表示文件类型或文件特性。

现代系统中，扩展名更多是**约定**而不是绝对依据。

## 引入文件的优点
### 1. 用户使用方便：实现按名存取
用户无需知道数据在辅助存储器中的物理位置，也无需关心数据如何存放到磁盘上，只需要知道文件名并发出操作请求即可。
### 2. 文件安全可靠
用户不能直接随意操作磁盘物理块，而是必须通过文件系统访问文件。文件系统可以提供：
```
权限控制
访问检查
文件保护
数据备份
错误恢复
```
这样可以防止用户有意或无意地破坏文件，也可以防止非法窃取文件内容。
### 文件可备份、可恢复
文件系统可以组织转储或备份，在硬件故障时提高可靠性

### 文件可共享
文件系统可以允许不同用户使用同一个文件，可以是同名共享，也可以是异名共享。

共享可以提高文件和存储空间的利用率。

## 文件系统及其功能
文件系统是操作系统中负责存取和管理信息的模块。它用统一的方式管理用户和系统信息的存储、检索、更新、共享和保护，并为用户提供一整套方便有效的文件使用和操作方法。

### 文件系统中的“文件”有两个视角
同一个文件必须从两个侧面观察：**逻辑文件**和**物理文件**。

**逻辑文件**
逻辑文件是用户看到的文件。

**物理文件**
物理文件是文件在辅助存储器上的实际存放形式。

### 文件系统面向用户的功能

**文件的按名存取**
这是文件系统最基本的功能。
用户通过文件名或路径名访问文件

文件系统需要完成：
```
解析路径 /home/user/a.txt
查找根目录 /
进入 home 目录
进入 user 目录
找到 a.txt 对应的目录项
找到文件控制块或 inode
定位文件数据块
完成读写
```

用户只看到文件名，系统内部则需要进行目录检索、元数据读取、权限检查和物理块定位。
文件名 / 路径名
        ↓
目录项
        ↓
FCB / inode
        ↓
物理块地址
        ↓
磁盘 I/O

### 文件的共享和保护
文件共享是指多个用户或多个进程可以访问同一个文件。
但共享必须伴随保护。否则，多个用户都能随意修改文件，会导致数据破坏。

常见保护机制包括：
```
访问控制矩阵
访问控制表 ACL
用户/组/其他用户权限
读/写/执行权限位
文件属性
加密
备份和恢复
```

### 文件的操作和使用
文件系统要为用户和程序提供一组操作接口。
用户程序看到的是简单接口，文件系统内部却要完成目录查找、权限验证、缓冲区管理、物理块读取等复杂工作。

### 为了实现这些功能，操作系统必须考虑什么？
#### 1. 文件目录的建立和维护
目录的作用是建立：
`文件名 → 文件元数据 / 文件位置`
的映射关系。
#### 2. 存储空间的分配和回收
文件创建或扩展时，文件系统要为它分配磁盘空间；文件删除或截断时，文件系统要回收空间。

#### 3. 数据的保密和保护
保密强调防止未授权读取，保护强调防止破坏、误改、误删。

#### 4. 监督用户存取和修改文件的权限
每次文件访问都需要检查权限。

#### 5. 处理不同存储介质的差异
文件可能存储在磁盘、SSD、光盘、磁带、网络存储等介质上。
文件系统要在不同介质上统一提供类似的文件接口。

# 文件的组织
**文件组织到底在解决什么问题**？
用户看到的是“有名字、有意义的数据对象”，比如 `student.dat`、`main.c`、`report.pdf`；  
磁盘看到的是“一个个固定大小或近似固定大小的物理块”。

```
用户看到的文件内容
    ↓
逻辑结构：字节流 / 记录流
    ↓
成组与分解：逻辑记录如何装入物理块
    ↓
物理结构：文件块在磁盘上如何排列
    ↓
真正的磁盘块 / 扇区 / 簇
```

## 文件的存储
### 文件存储介质：磁带、光盘、磁盘
文件存储介质主要有 `磁带、光盘、磁盘`
其中磁带和光盘更偏向顺序存取，磁盘是典型的直接存取或随机存取设备。

**卷**和**块**
卷是存储介质的物理单位，例如一盘磁带、一张光盘、一个硬盘分区；
块是存储介质上一段连续信息组成的区域，也叫物理记录，并且块是主存和辅存之间交换信息的物理单位

|概念|含义|例子|
|---|---|---|
|存储介质|保存文件的物理设备或材料|磁盘、磁带、光盘、SSD|
|卷|文件系统管理的存储介质单位|一个硬盘分区、一个光盘片|
|块|主存和辅存交换数据的基本物理单位|4KB 块、8KB 块|
|物理记录|存储介质上一块连续信息|通常可以理解为块|

### 块大小为什么重要？
决定块大小时要考虑用户使用方式、数据传输效率、存储设备类型等因素；相邻块之间还可能因为机械动作或块识别需要留有间隙，间隙是不记录用户代码信息的区域。

块大小是文件系统设计中的典型“时间—空间折中”。

**块大一点的好处**
```
一次 I/O 传输的数据更多
顺序读写吞吐量更高
文件块索引数量减少
元数据开销减少
磁盘寻道和旋转延迟被摊薄
```
适合大文件、视频文件、备份文件、日志归档文件。

**块大一点的问题**
```
小文件浪费空间
随机访问时可能读入很多无用数据
缓存污染可能更明显
```

**块小一点的好处**
```
小文件空间浪费较少
随机访问粒度更细
```

**块小一点的问题**
```
大文件需要更多块
索引表更大
I/O 次数可能增多
管理开销更高
```
所以块大小不是越大越好，也不是越小越好，而要根据文件系统目标和设备特性折中。

### 顺序存取存储设备的信息安排
> 顺序存取设备是严格依赖信息的物理位置次序进行定位和读写的存储设备。

磁带机是典型顺序存取设备，优点是容量大、可靠、卷可装卸、便于保存，常用于归档。磁带的块长变化范围较大，块可以很小，也可以很大。
光盘也可作为顺序存取存储设备来理解。

顺序存取设备的特点是：
```
读第 n 个块之前，往往要经过前面的块
适合批量顺序读写
不适合频繁随机访问
```

典型场景
```
备份
归档
批处理输入文件
顺序扫描的大型数据文件
```

从性能角度看，顺序存取设备强调“按物理顺序组织信息”。如果你频繁跳来跳去访问，会非常慢。

### 直接存取存储设备的信息安排
磁盘是一种直接存取存储设备，也叫随机存取存储设备；它通过移臂和旋转这二维组织实现较高存取速度，并且每个物理记录有确定位置和唯一地址，访问任意物理块的时间几乎不依赖于该信息的位置。

这里“几乎不依赖”要理解为操作系统教材中的抽象说法。实际磁盘仍然有：
```
寻道时间
旋转延迟
传输时间
```

所以磁盘随机访问比顺序访问慢很多，但比磁带随机定位强得多。对于 SSD 来说，没有机械寻道和旋转延迟，随机访问性能进一步提升，但仍然存在页、块、擦除、写放大等底层约束。

直接存取设备的意义是：文件系统可以更灵活地选择文件物理结构，比如连续分配、链接分配、索引分配、哈希直接定位等。

## 文件的逻辑结构

**什么是逻辑文件**？
逻辑文件也称为文件的逻辑结构，它是独立于物理环境的、用户概念中的抽象信息组织方式，是用户能观察并处理的数据集合。文件逻辑结构主要分为两种：**流式文件**和**记录式文件**。

注意，逻辑结构不关心文件在磁盘上连续还是分散，也不关心数据块号是多少。那些是物理结构的问题。

### 流式文件
> 流式文件指文件内的数据不再组成记录，只是由一串依次的字节组成的信息流序列；这种文件常常按长度读取所需信息，也可以用插入的特殊字符作为分界。

`文件 = byte[0], byte[1], byte[2], ..., byte[n-1]`
用户程序通过“偏移量 + 长度”访问文件。

**流式文件的特点**

|方面|特点|
|---|---|
|结构|字节流|
|访问方式|按字节偏移和长度访问|
|灵活性|很强|
|操作系统负担|较小，不需要理解记录格式|
|典型系统|Unix/Linux 普通文件|
|典型文件|`.txt`、`.c`、`.jpg`、`.mp4`、`.bin`|
很多现代操作系统倾向于把普通文件看成字节流。至于字节流里面是不是 JSON、CSV、图片、音频、数据库页，那通常由应用程序解释。

### 记录式文件
> 记录式文件是一种有结构的文件，它是若干逻辑记录信息组成的记录流文件；逻辑记录是文件中按信息在逻辑上的独立含义所划分的信息单位。

例如工资文件：
```
记录1：00001 张三 计算机系 76.80 ...
记录2：00002 李四 计算机系 105.00 ...
记录3：00003 王五 物理系 123.60 ...
```
记录式文件可以理解为：
文件 = 记录1 + 记录2 + 记录3 + ... + 记录n
记录 = 数据项1 + 数据项2 + ... + 数据项m

#### 固定长度记录与可变长度记录
记录式文件还可以继续细分。

**固定长度记录**
每条记录长度相同。

优点是定位简单：
```
第 i 条记录的偏移 = 文件起始地址 + i × 记录长度
```
固定长度记录适合直接定位，但可能浪费空间，因为不同记录实际内容长度可能不同。

**可变长度记录**
每条记录长度不一定相同。

优点是节省空间，适合文本、日志、复杂对象；缺点是定位困难，因为要知道前面所有记录长度，或者额外建立索引。

**记录式文件和数据库的区别**
数据库管理系统也支持逻辑记录，但数据库不同于记录式文件。数据库中的记录之间可以通过数据冗余构成某种联系，数据库管理系统支持基于联系的数据查询，而文件系统不支持这种查询。

## 记录的成组与分解
一个物理记录只存放一个逻辑记录可能造成极大浪费；若干个逻辑记录合并成一组，写入一个块，叫记录的成组；每块中的逻辑记录数称为块因子。

逻辑记录：用户意义上的一条记录
物理记录：存储介质上的一个块

如果一个逻辑记录很小，而一个物理块很大，那么“一块只放一条记录”会浪费大量空间。


### 记录成组是什么？
多个逻辑记录 → 合并 → 一个物理块

对于流式文件，指出一个物理记录可以存放很多个连续字节。
### 记录分解是什么？
当存储介质上的一个物理记录读进输入缓冲区后，把逻辑记录从块中分离出来的操作叫记录的分解操作

一个物理块 → 拆出 → 多条逻辑记录

这说明用户请求一条记录时，系统底层可能会把包含它的整个物理块读入内存，然后从中取出目标逻辑记录。

### 成组与分解在哪里完成？
系统设置独立于用户数据区的输入/输出缓冲区；记录成组在输出缓冲区内进行，凑满一块后才写到存储介质；读入时，一个物理记录先进入输入缓冲区，再从块中分离出逻辑记录。

### 非跨块记录与跨块记录
两种可变长度记录成组方式：**可变长度非跨越式组块**和**可变长度跨越式组块**

**可变长度非跨越式组块**
记录长度可变
但一条记录不能跨越两个物理块
如果当前块剩余空间放不下下一条记录，剩余空间就浪费

**优点**
记录读取简单
一条记录一定完整存在于一个块中

**缺点**
块内可能出现未使用空间
空间利用率下降

**可变长度跨越式组块**
记录长度可变
允许一条记录跨越两个或多个物理块
块中尽量不留空闲空间
跨块部分通过指针或描述信息连接

**优点**
空间利用率高
减少块内浪费

**缺点**
记录读取复杂
一条记录可能需要读多个块
需要维护跨块连接信息
如果某个块损坏，恢复更麻烦

### 成组与分解带来的新特征：提前读与推迟写
用户读请求会导致包含目标逻辑记录的整个物理块被读入输入缓冲区，这可能同时读入多个逻辑记录，称为提前读；用户写请求首先写入输出缓冲区，只有缓冲区中的逻辑记录满后才会实际输出，称为推迟写。

#### 提前读 read-ahead
用户只想读 R3，但系统读入整个块：
好处是，如果用户接下来读取 R4，就不用再访问磁盘。

#### 推迟写 delayed write
用户写入 R1、R2、R3 时，系统可能先放在输出缓冲区，不立即写盘：
好处是减少 I/O 次数，提高性能。
风险是：如果系统崩溃，缓冲区中尚未落盘的数据可能丢失。

## 文件的物理结构
文件的物理结构和组织是指文件在物理存储空间中的存放方法和组织关系，又称物理文件。文件的存储结构涉及块的划分、记录的排列、索引的组织、信息的搜索等问题，其优劣直接影响文件系统性能。

物理结构直接影响：
顺序访问速度
随机访问速度
空间利用率
文件扩展是否方便
插入删除是否方便
元数据开销
可靠性

### 顺序文件，也叫连续文件
将一个文件中逻辑上连续的信息存放到存储介质的依次相邻的块中，便形成顺序结构，这类文件叫顺序文件，又称连续文件。磁带文件、光盘文件是典型例子。

**优点**
顺序文件在顺序存取记录时速度较快，批处理文件、系统文件用得最多；如果用磁带存放顺序文件，总能保持快速存取的优点。

**缺点**
建立文件前需要预先确定文件长度，以便分配存储空间；修改、插入、增加文件记录比较困难。

### 连接文件，也叫串联文件
连接结构使用连接字表示文件中各个物理块之间的先后次序。第一块文件信息的物理地址由文件目录给出，每一块的连接字指出文件的下一个物理块位置；连接字为 0 时表示文件到本块结束。

```
目录项：
文件 A 首块 = 9

磁盘块：
9  → 16 → 3 → 25 → 0
A0   A1   A2  A3   结束
```
每个块除了保存数据，还保存一个指针：
```
当前块数据 + 下一块地址
```

**优点**
文件扩展方便
插入删除相对方便
外部碎片问题小
空闲块利用率高
适合动态增长文件

**缺点**
随机访问差
连接字占用空间
指针损坏可能导致后续文件块丢失
不利于磁盘预读和连续传输
磁头可能频繁移动

### 直接文件
直接文件也常被称为散列文件或哈希文件。它的思想是：
根据记录关键字直接计算出记录所在的物理地址

**优点**
按关键字访问速度很快
平均情况下接近 O(1)
适合需要快速查找单条记录的场景

**问题**
冲突
不适合顺序遍历
不适合范围查询
哈希函数设计困难
文件扩展可能导致重散列
空间利用率可能不稳定

### 索引文件
索引文件在文件存储器上分为两个区：
```
索引区
数据区
```
访问索引文件需要两步：第一步查找索引表，第二步获得记录物理地址；如果文件索引已经预先调入主存，可以减少一次内外存信息交换。

```
索引表：
逻辑块0 → 物理块 12
逻辑块1 → 物理块 98
逻辑块2 → 物理块 31
逻辑块3 → 物理块 77

数据区：
12 保存 A0
98 保存 A1
31 保存 A2
77 保存 A3
```

**优点**
索引结构可以看作连接结构的扩展；它具备连接文件的优点，同时克服了连接文件只能顺序存取的缺点，能够直接读写任意记录，便于文件增、删、改。

**缺点**
增加了索引表的空间开销和查找时间。
索引块本身占空间
小文件也可能需要索引结构
访问数据前可能要先访问索引
多级索引会增加寻址次数
索引损坏可能影响整个文件访问

### 一级索引、两级索引、多级索引
```
一级索引块
 ├─ 二级索引块A
 │    ├─ 数据块地址
 │    ├─ 数据块地址
 │    └─ ...
 ├─ 二级索引块B
 │    ├─ 数据块地址
 │    └─ ...
 
 一级索引 → 二级索引 → 三级索引 → 数据块
```
优点是可以支持非常大的文件；缺点是访问深层数据块时需要更多次索引访问。

**假设**
```
块大小 = 1KB
块地址 = 4B
一个索引块可存 256 个地址

一级间接索引可索引 256 个数据块
二级间接索引可索引 256 × 256 个数据块
三级间接索引可索引 256 × 256 × 256 个数据块
```
