[[操作系统]]

如何把用户看到的地址空间，安全、高效、灵活地映射到政治贮存甚至辅存上

**逻辑地址** 用户编程时使用的地址空间
逻辑地址从0开始编号，用两种形式：
一维逻辑地址，即普通“地址”
二维逻辑地址，即“段号：段内地址”

**物理地址** 程序真正执行时访问主存所使用的地址空间
CPU执行指令时最终按照物理地址访问主存

## 存储管理的主要模式
四类：单连续存储管理、段式存储管理、页式存储管理、段页式存储管理

主要差别：
- 程序段逻辑地址空间是一维还是二维
- 主存空间是按连续分区分配，还是按固定大小页框分配

### 主存复用的两种基本方式
多道程序设计要求多个程序共享主存

**按照分区复用**
将主存划分成多个固定或可变大小的分区，一个程序或程序段占用一个分区

**按照页框复用**
把主存划分成多个固定大小的页框，一个程序或程序段占用多个页框

### 单连续存储管理
**单连续存储管理**是最简单的模式
定义为：一维逻辑地址空间的程序，占用一个主存固定分区或可变分区

特点：一个进程的代码、数据、堆栈等整体被装入一个连续的主存区域
```
逻辑地址空间：
0 -------------------------------- n

主存：
[ 操作系统 ][ 进程A连续区 ][ 进程B连续区 ][ 空闲区 ]
```

优点：
简单，地址转换容易
`物理地址 = 基址 + 逻辑地址`

缺点：
进程必须放在一整块连续空间中
如果内存中虽然总空闲空间足够，但被分散成很多小块，就可能放不下一个大进程，这就是外部碎片问题

### 页式存储管理
**页式存储管理**适用于一维逻辑地址空间

一维逻辑地址空间大成组，占用多个主存页框，也叫页架

页式管理的核心思想：把进程的逻辑地址空间切成固定大小的“页”，把物理主存切成同样大小的“页框”。进程的各个页不必连续存放，可以分散装入不同页框
