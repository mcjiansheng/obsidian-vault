[[操作系统]]
> CPU 很快，I/O 设备很慢；设备类型又多又杂。操作系统必须屏蔽硬件差异，让用户程序用统一接口访问设备，同时尽量让 CPU、内存、外设并行工作。

I/O 设备的定义:
I/O 设备又称输入输出设备、外围设备、外设，用于计算机系统与外部世界进行信息交换或存储；I/O 操作本质上就是**内存与外设之间的信息传送**。这些设备会直接影响系统通用性、扩充性、综合处理能力和性价比。

## 输入输出 I/O 管理
### I/O 设备的分类
第一，从**信息传输方向**看：
输入设备把外界信息输入计算机，比如键盘、鼠标、扫描仪；输出设备把计算结果输出，比如显示器、打印机；输入输出设备既能输入又能输出，比如磁盘驱动器、网卡。

第二，从**交互功能**看：
人机交互设备用于人与计算机交互，比如鼠标、键盘、显示器；存储设备用于大量信息存储和快速检索，比如磁盘、光盘；机机通信设备用于计算机之间通信，比如网卡、调制解调器。

第三，从**设备管理方式**看：
字符设备以字符为单位交换信息，例如键盘、鼠标、显示器；块设备以固定大小的数据块为单位交换信息，例如磁盘；网络设备用于机器通信，既可以抽象为字符流设备，也可以抽象为连续小块数据传送设备。

### 设备管理的目标
第一，解决**设备与 CPU 速度不匹配**的问题，让主机和设备充分并行工作，提高设备使用效率。比如 CPU 不应该一直等打印机慢慢打印，也不应该每读一个字节就被磁盘拖住。

第二，屏蔽设备的物理细节和操作过程，通过驱动程序提供统一界面。课件中特别提到，操作系统可以把设备抽象成**裸设备**，也可以抽象成**设备文件**。

### 设备管理的功能
设备中断处理、缓冲区管理、设备的分配和去配、设备驱动调度、虚拟设备的实现。

用户程序提出 I/O 请求；操作系统检查权限和设备状态；如果是独占设备，要分配设备；如果设备慢，要使用缓冲；如果设备有多个请求，要调度；真正传输时由驱动、控制器、DMA 或通道完成；完成后设备发出中断；操作系统处理中断并唤醒等待进程；如果设备不存在或很慢，还可以通过 SPOOLing 等虚拟设备技术进行模拟。

## I/O 硬件基础：设备控制器、DMA、通道
### 设备控制器
设备通常由机械部件和电子部件组成。课件指出，为了模块化和通用性，系统通常把设备的机械部件和电子部件分开，电子部件称为**设备控制器**，也叫设备适配器、I/O 控制器、I/O 控制接口、I/O 模块。系统实际上是和控制器交互，而不是直接和物理设备交互。

设备控制器的主要功能包括：

接收和识别 CPU 或通道发来的命令；实现数据交换；发现和记录设备及控制器自身的状态信息；当一个控制器连接多台设备时，还要识别设备地址。

控制器一般包含数据缓冲寄存器、状态/控制寄存器、地址译码与 I/O 控制逻辑。CPU 可以通过这些寄存器向设备发命令、读状态、传输数据。

### I/O 控制方式
常见方式有：轮询方式、中断方式、DMA 方式、通道方式。

#### 轮询方式
轮询方式也叫程序直接控制方式。CPU 向控制器发送 I/O 命令，然后不断读取控制器状态。如果设备未就绪，就反复测试；设备就绪后，CPU 执行内存和设备之间的数据交换。
这种方式下 CPU 在等待 I/O 期间不能有效执行其他工作。

优点是实现简单，适合非常简单的嵌入式系统或低速设备。缺点是浪费 CPU 时间，系统并行性很差。
#### 中断方式
CPU 向控制器发出 I/O 命令后继续执行其他指令；当设备准备好或 I/O 完成时，控制器向 CPU 发出中断；CPU 转入中断处理程序，完成后续处理。
如果进程支持异步 I/O，CPU 可以继续执行该进程后续指令；否则，该进程在这个 I/O 上挂起，CPU 执行其他工作。

中断方式比轮询方式好，因为 CPU 不必一直忙等。但它仍然有一个问题：数据传输过程仍可能需要 CPU 参与，尤其是大量数据传输时，中断次数和数据搬运开销会很大。

#### DMA 方式
DMA 是 Direct Memory Access，直接存储器访问。
DMA 模块能够替代处理器来控制主存和设备控制器之间的数据交换。
CPU 向 DMA 模块发出 I/O 命令；CPU 继续执行其他工作；DMA 模块负责传送全部数据；数据传送结束后，DMA 中断处理器。

DMA 的关键价值是：**CPU 不再逐字节搬运数据**。CPU 只负责初始化 DMA，例如设置内存地址、传输长度、传输方向、设备号；真正的数据搬运由 DMA 控制器完成。

DMA 方式适合磁盘、网卡、显卡等高速块传输设备。

#### 通道方式
I/O 通道可以看成更强的 I/O 处理器。用于完成逻辑上独立的 I/O 任务；

系统采用“处理器—通道—控制器—设备”的四级连接；CPU 不再执行 I/O 指令，而是在主存中组织通道程序，由 I/O 通道执行。

通道方式的流程是：CPU 遇到 I/O 任务后组织通道程序，设置通道程序地址字 CAW，启动指定通道；通道从 CAW 获得通道程序，控制 I/O 设备操作，CPU 执行其他任务；I/O 完成后，通道发出中断，CPU 根据通道状态字 CSW 处理结果。

|控制方式|CPU 是否等待|数据传输由谁做|适用场景|
|---|--:|---|---|
|轮询|等待/忙等|CPU|简单低速设备|
|中断|不一直等待|CPU 仍参与较多|字符设备、中低速 I/O|
|DMA|基本不参与搬运|DMA 控制器|磁盘、网卡等块设备|
|通道|CPU 只发起任务|通道执行通道程序|大型机/复杂 I/O 系统|

## I/O 软件层次

I/O 管理不能只看硬件。操作系统真正厉害的地方在于：它用软件层次把不同设备封装成统一接口。

设备管理实现层次包括：I/O 硬件、I/O 设备及其接口线路、控制部件、通道、I/O 软件、系统 I/O 软件、用户空间 I/O 软件。

I/O 软件通常可分为四层：
1. 用户层 I/O 软件
2. 设备独立性软件
3. 设备驱动程序
4. 中断处理程序
从上往下越来越接近硬件，从下往上越来越接近用户接口。
### 用户层 I/O 软件
用户层 I/O 软件包括库函数和一部分虚拟设备软件。
一部分 I/O 软件可以用库函数实现，放在操作系统内核之外，运行时与应用程序链接；虚拟设备软件则用一类设备模拟另一类设备。

例如 C 语言中的 `printf()`、`scanf()`、`fread()`、`fwrite()` 本身不是内核函数，而是库函数；最终它们会调用系统调用，如 `read()`、`write()`。

### 设备独立性软件
设备独立性软件负责所有设备通用的 I/O 功能，并向用户层提供一致接口。
包括：设备命名、设备保护、提供与设备无关的数据单位、缓冲技术、分配和状态跟踪、错误处理/报告。

这一层非常重要。它实现了“用户不关心具体物理设备”的目标。
### 设备驱动程序
设备驱动程序是和具体硬件绑定的内核模块。它知道某个设备控制器的寄存器含义、命令格式、中断号、DMA 设置方式。

比如磁盘驱动程序知道如何向磁盘控制器提交读写请求；网卡驱动知道如何配置收发队列；显卡驱动知道如何提交图形命令。

驱动程序向上提供统一接口，向下操作具体硬件，是“设备无关接口”和“设备相关硬件”之间的桥。

### 中断处理程序
中断处理程序负责响应设备中断。设备完成 I/O 后，控制器发出中断；CPU 保存现场，转入中断处理程序；中断处理程序检查设备状态、处理错误、更新 I/O 请求状态、唤醒等待进程。

## I/O 应用接口
从用户程序角度看，I/O 接口主要有几类：

第一，**字符设备接口**。按字节流读写，例如键盘、终端、串口。典型操作是 `read()`、`write()`。
第二，**块设备接口**。按块随机访问，例如磁盘。文件系统通常建立在块设备之上。
第三，**网络设备接口**。通过 socket 等接口访问，关注数据包、连接、端口等抽象。
第四，**控制接口**。例如 Unix/Linux 中的 `ioctl()`，用于设置设备参数，如终端属性、磁盘参数、网卡模式等。

还要区分三组概念：
**阻塞 I/O**：请求发出后，进程等待 I/O 完成。  
**非阻塞 I/O**：请求立即返回，如果数据没准备好就返回错误或特殊状态。  

**异步 I/O**：请求发出后进程继续运行，I/O 完成后系统通知进程。  
**同步 I/O**：调用方需要在某个点等待 I/O 结果。  

**缓冲 I/O**：数据先进入内核缓冲区，再择机写入设备。  
**直接 I/O**：尽量绕过页缓存或缓冲区，直接在用户缓冲区和设备之间传输。

应用接口的根本目标是**统一、抽象、保护、效率**。
## I/O 缓冲区与高速缓存

### 为什么需要 I/O 缓冲？
设置 I/O 缓冲的目的：
解决 CPU 与设备速度不匹配；
协调逻辑记录大小和物理记录大小不一致；
提高 CPU 和设备的并行性；
减少 I/O 操作对 CPU 的中断次数；
放宽对 CPU 中断响应时间的要求。

> 缓冲区的本质是用内存空间换时间、换并行性、换接口一致性。
### I/O 缓冲区的定义
I/O 缓冲区是在内存中开辟的存储区，专门用于临时存放 I/O 操作的数据。写操作时，数据先送入缓冲区，装满或需要写出时再写到设备；读操作时，系统先把设备上的物理记录读到缓冲区，再按进程需要把数据传给进程。

### 单缓冲、双缓冲、循环缓冲
**单缓冲**：系统区开一个缓冲区。输入时，先把数据读入缓冲区，再从缓冲区传到用户区；输出时，先把用户数据放入缓冲区，再输出到设备。实现简单，但并行度有限。

**双缓冲**：系统区开两个缓冲区。当应用程序处理缓冲区 1 的数据时，设备可以同时向缓冲区 2 输入数据；输出时也类似。双缓冲提高了设备和进程的并行性。

**循环缓冲**：系统分配一组缓冲区，每个缓冲区有指向下一个缓冲区的指针，构成环形结构，用于继续调节设备和进程速度不匹配的问题。
### 缓冲区和高速缓存的区别
**缓冲区 buffer** 主要解决速度不匹配、数据粒度不匹配、生产者消费者节奏不一致的问题。它强调“临时中转”。

**高速缓存 cache** 主要利用局部性原理，把访问过或可能访问的数据保存在更快的存储层次中。它强调“复用”。
## 设备分配与回收
设备按照共享方式可以分为三类：
**独占设备**：一次只能被一个进程使用，比如传统打印机、磁带机。  
**共享设备**：多个进程可以交替访问，比如磁盘。  
**虚拟设备**：用共享设备模拟独占设备，比如用磁盘上的输出井模拟打印机。

**设备独立性**
如果作业申请时指定具体物理设备，虽然分配简单，但如果该设备故障，即便系统中有同类设备也不能运行。
设备独立性的做法是用户指定逻辑设备，由系统建立逻辑设备和物理设备之间的映射。

设备独立性的优点是：应用程序与具体物理设备无关；系统增减或变更设备时不需要修改程序；更容易处理设备故障；增加设备分配灵活性，有利于提高设备利用率和实现多道程序设计。

独占设备分配方式有两类：
**静态分配**：进程运行前申请所需设备。优点是实现简单、可以防止死锁；缺点是设备利用率低。

**动态分配**：进程运行过程中随用随申请。优点是提高设备利用率；缺点是可能引入死锁，需要结合互斥、同步、资源分配策略处理。

## SPOOLing 假脱机技术
SPOOLing 是 Simultaneous Peripheral Operations On-Line 的缩写，中文常称为**外部设备联机并行操作**或**假脱机技术**。它是关于慢速字符设备如何与计算机主机交换信息的一种技术。

> SPOOLing 是一种用共享型磁盘设备模拟独占型物理设备的技术，也是一种速度匹配技术。

典型例子是打印机。打印机本来是独占设备：一个时刻只能打印一个作业。如果多个进程直接争用打印机，就会阻塞、混乱、效率低。SPOOLing 让进程先把输出写入磁盘上的**输出井**，系统后台再由缓输出程序慢慢送到打印机。

它的好处是：
1. 把慢速独占设备虚拟成可共享设备。
2. 缩短进程在内存中的等待时间。
3. 提高系统吞吐量。
4. 让作业输入输出与作业计算重叠执行。

## 磁盘组织管理
### 磁盘为什么特殊？
第一，它是**块设备**，以固定大小的数据块为单位传输。  
第二，它是**直接存取设备**，可以按地址访问任意块。  
第三，它有明显的机械访问代价，尤其是寻道时间和旋转延迟。

所以磁盘既属于设备管理，又是文件系统、虚拟内存、交换区的底层基础。

### 磁盘物理结构
磁盘一般由多个盘片组成；每个盘片一般有两个盘面；盘面包括多个同心圆结构的磁道；不同盘面上相同位置的磁道构成柱面；每个磁道分成多个固定扇区；相邻扇区可以组合成簇。物理块地址可以用“柱面号、磁头号、扇区号”表示。

**盘片 platter**：磁盘中的圆形存储介质。  
**盘面 surface**：一个盘片通常有上下两个盘面。  
**磁头 head**：每个盘面对应一个读写磁头。  
**磁道 track**：盘面上的同心圆。  
**柱面 cylinder**：不同盘面上半径相同的磁道集合。  
**扇区 sector**：磁道被划分成的固定小区域，是磁盘读写的基本物理单位之一。  
**簇 cluster**：文件系统为了管理方便，把若干扇区组成一个更大的分配单位。

### 磁盘读写过程
读写数据时，磁头必须定位到指定磁道上的指定扇区开始处，过程包括：寻道，控制移动臂到达指定柱面；旋转，等待要读写的扇区旋转到磁头下；选择磁头号，进行数据传送。

所以一次磁盘访问通常包括：
`等待设备/通道 → 寻道 → 旋转延迟 → 数据传送`

磁盘存取时间可以表示为：
$Ta = Ts + Tr + Tt$
`Ta` 是访问时间；  
`Ts` 是寻道时间；  
`Tr` 是旋转延迟；  
`Tt` 是传送时间。

如果磁盘转速为 `r` 转/秒，平均旋转延迟通常近似为：
$Tr = 1 / (2r)$

如果要传送 `b` 字节，一个磁道有 `N` 字节，则传送时间近似为：
$Tt = b / (rN)$

因此常见公式是：

$Ta = Ts + 1/(2r) + b/(rN)$

**磁盘访问慢，主要慢在寻道和旋转，而不是数据真正传输。**

## 磁盘调度算法
### 为什么需要磁盘调度？
磁盘可能同时接收到多个 I/O 请求；如果随机响应 I/O 请求，会得到很差的性能。磁盘驱动调度就是操作系统按照某种较优次序处理多个磁盘访问请求，以减少磁盘访问总处理时间。驱动调度策略包括移臂调度和旋转调度。

磁盘调度的核心目标是：
1. 减少磁头移动距离。
2. 减少平均寻道时间。
3. 提高吞吐量。
4. 控制响应时间。
5. 尽量避免饥饿。
6. 兼顾公平性。

### FCFS / FIFO：先来先服务
FCFS 按请求到达顺序处理。

优点是公平、简单、不会饥饿。

缺点是完全不考虑磁头当前位置，磁头可能来回大幅移动，平均寻道时间可能很长。
先来先服务移臂距离大，性能不好。

适合请求量小、系统负载低、对公平性要求高的场景，不适合高负载磁盘系统。

### SSTF：最短寻道时间优先
每次选择距离当前磁头位置最近的请求。
先执行查找时间最短的请求，寻道性能较好，但存在饥饿现象。

优点是平均寻道距离通常较小；缺点是远处请求可能长期得不到服务。尤其当磁头附近不断有新请求到来时，远端请求会被饿死。

### SCAN：扫描算法
SCAN 又叫电梯算法的基础版本。磁头沿一个方向移动，途中处理遇到的请求，到达一端后再反向移动。

扫描分为单向扫描和双向扫描：单向扫描中移动臂向一个方向扫描，归途不提供服务；双向扫描中移动臂每次向一个方向移动，遇到最近的 I/O 请求便处理，到达最后柱面后再向反方向移动。

SCAN 的优点是比 FCFS 性能好，比 SSTF 更公平，不容易让远端请求无限等待。缺点是两端柱面的等待时间可能不均匀，中间区域可能得到更多服务机会。

### LOOK / 电梯调度
是双向扫描的改进：当前移动方向没有访问请求时，就改变移动方向。

这其实接近教材中常说的 LOOK 算法。SCAN 会一直走到磁盘末端再返回；LOOK 不一定走到最边界，而是“看到”当前方向已经没有请求，就掉头。

LOOK 更节省无意义移动，实际系统更常用类似策略。

### C-SCAN：循环扫描
C-SCAN 把扫描限定在一个方向。当磁头访问到某方向最后一个磁道后，直接返回到相反方向末端，再重新开始扫描。
C-SCAN 在访问到沿某方向的最后磁道时，磁头臂返回到磁盘相反方向磁道的末端，并再次开始扫描。

C-SCAN 的优点是等待时间更均匀，适合请求分布较均匀、负载较高的系统。缺点是回程不服务请求，会产生额外移动。

## 旋转调度与数据优化分布
磁盘调度不只有移臂调度。

**旋转调度**
旋转调度的目标是使旋转延迟的总时间最少。
“循环排序”：通过优化 I/O 请求排序，在最少旋转圈数内完成位于同一柱面的访问请求；旋转位置测定硬件和多磁头同时读写技术有利于提高旋转调度效率。

此外还有**优化分布**。
可以通过信息在存储空间中的排列方式减少旋转延迟，例如交替排序。原因是磁盘匀速旋转，处理当前扇区数据时，下一个连续编号扇区可能已经转过去了。通过扇区交叉编号，可以让系统处理完当前扇区后，刚好等到下一个逻辑扇区转到磁头下。

还可以把相邻扇区集中成簇读写，或者按柱面集中存储数据，从而减少移臂操作。

## 磁盘管理：格式化、分区、坏块、引导块、空闲空间
### 低级格式化
低级格式化把磁盘划分为扇区，为每个扇区建立头部、数据区、校验信息等结构。它面向物理介质，解决“磁盘表面如何被组织成可读写扇区”的问题。
现代硬盘的低级格式化通常由厂商完成，普通用户很少真正执行底层低级格式化。
### 分区
分区是把一个物理磁盘划分成若干逻辑区域。每个分区可以独立建立文件系统，也可以作为交换分区、恢复分区、EFI 系统分区等。
从 OS 视角看，分区让一个物理磁盘变成多个逻辑卷，便于安装不同文件系统和进行空间管理。
### 逻辑格式化
逻辑格式化是在分区上建立文件系统，例如 FAT、NTFS、ext4。它会建立超级块、空闲空间管理结构、inode 区、目录区等元数据。
EXT2 会把磁盘逻辑分区划分为块组，每个块组包括超级块、组描述符表、块位示图、inode 位示图、inode 表以及数据块。
### 引导块
引导块用于保存启动操作系统所需的引导信息。计算机启动时，固件会找到启动设备上的引导信息，加载引导程序，再加载操作系统内核。
### 坏块管理
磁盘可能出现坏块。操作系统和磁盘控制器可以通过坏块表、备用扇区重映射等方法处理。对用户和文件系统来说，理想情况是坏块被屏蔽，不再分配给文件。
### 空闲空间管理
文件系统必须知道哪些磁盘块空闲、哪些已经占用。常见方法包括：
- 位示图：每个磁盘块用 1 bit 表示状态。  
- 空闲链表：把空闲块串起来。  
- 成组链接法：UNIX/Linux 早期常用，将空闲块分组管理。  
- 计数法：记录连续空闲块的起始地址和长度。

## RAID 磁盘冗余阵列

RAID 的核心思想是：把多个磁盘组织成一个逻辑磁盘，通过**条带化、镜像、校验**提高性能和可靠性。

**RAID 0：条带化**。数据分散到多个磁盘并行读写，性能高，但没有冗余，一个盘坏可能导致数据丢失。

**RAID 1：镜像**。两个或多个磁盘保存相同数据，可靠性高，读性能可能提升，写入成本和空间成本高。

**RAID 4：专用校验盘**。数据分条带存储，使用一个专门磁盘保存校验信息。缺点是校验盘容易成为写瓶颈。

**RAID 5：分布式校验**。校验信息分布在多个磁盘上，避免专用校验盘瓶颈，兼顾空间利用率和可靠性。

**RAID 10：先镜像再条带化**。兼顾性能和可靠性，但成本高。
