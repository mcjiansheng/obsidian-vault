## 传统单线程进程

在早期操作系统或传统进程模型中，一个进程通常只有一条执行路径，也就是 单线程结构进程
包括：
```
进程控制块 PCB
用户地址空间
用户栈
核心栈
一条执行序列
```
也就是说，一个进程既是 **资源分配单位** ，又是 **CPU调度执行单位**

### 单线程结构的问题

```
进程切换开销大
进程通信开销大
限制进程并发粒度
降低并行计算效率
```

进程切换需要保存和恢复大量上下文，包括：
```
PC 
PSW
通用寄存器
栈指针
PCB 状态
地址空间信息
页表 / 内存映射信息
打开文件表相关信息
```
且不同进程通常有不同地址空间，切换时可能影响TLB、Cache等硬件状态
因此进程切换相对昂贵
