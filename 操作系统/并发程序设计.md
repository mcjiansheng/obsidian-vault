# 并发程序/并发进程
**提高系统资源利用率和系统吞吐率**

**把一个任务拆成若干个能重叠推进的进程，使 CPU、I/O 设备、内存等资源尽可能同时工作。**

### 顺序程序设计：并发的对照物
并发程序破坏了很多“默认安全假设”。

顺序程序设计中，程序是实现算法的操作或指令序列，每个程序在处理器上严格按序执行，这叫程序执行的**内部顺序性**；而把一个具体问题设计成一个程序，或者设计成若干严格顺序执行的程序序列，这叫程序执行的**外部顺序性**。

顺序程序设计有四个重要特性：
- **顺序性**：指令严格按程序规定的顺序执行。
- **封闭性**：程序运行时好像独占系统资源，外界不会随便干扰它。
- **确定性**：同一输入下，结果与执行速度、执行时间段无关。
- **可再现性**：同一输入下，执行轨迹可重复。

### 进程的并发执行
多道程序设计允许多个程序同时进入内存，它们竞争处理器，获得运行机会。操作系统允许在一个时间段内存在多个正在运行的进程，这就是**多个进程并发执行**。
对于按照顺序程序设计思想编写、且互不干扰的进程，操作系统应当保证它们并发执行时仍然像独占计算机一样不受影响，这类进程称为**无关的并发进程**。

**并发 Concurrency**：多个进程在时间上重叠推进。单处理器系统中，宏观上看多个进程都在运行，微观上任一时刻只有一个进程占用 CPU。
**并行 Parallelism**：多个进程在物理上真正同时执行。通常需要多核 CPU 或多处理器支持。

### 并发程序设计的概念
把一个程序分成若干个可同时执行的程序模块，每个程序模块与其处理的数据共同组成一个进程

```c
// 进程 i：输入
while (true) {
    input;
    send;
}

// 进程 p：处理
while (true) {
    receive;
    process;
    send;
}

// 进程 o：输出
while (true) {
    receive;
    output;
}
```
这里的 `send` 和 `receive` 表示进程之间需要通信或同步。输入进程读完数据后，处理进程才能处理；处理进程处理完后，输出进程才能输出。它们虽然并发执行，但不是完全随意执行，而是存在**制约关系**。

并发程序设计的三个特性是：
- **并行性 / 并发性**：多个进程可重叠执行，提高效率。
- **共享性**：多个进程共享软件资源或硬件资源。
- **交往性**：多个进程之间存在制约，需要协调。
**并发不是简单地“同时跑”，而是“同时推进但要协调”。**

## 并发进程的制约关系
并发进程分为两类：**无关的并发进程**和**交往的并发进程**。

**无关的并发进程**
无关的并发进程分别在不同变量集合上运行，一个进程的执行不会影响另一个进程的结果。

**交往的并发进程**
交往的并发进程共享某些变量或资源，一个进程的执行可能影响其他进程的结果。

### Bernstein 条件：判断两个程序能否安全并发
设 $R(p_i)$ 表示程序 $p_i$ 执行期间**引用/读取**的变量集合
$W(p_i)$ 表示程序 $p_i$ 执行期间 **改变/写入**的变量集合
两个程序 $p_1$​、$p_2$​ 若满足：
$(R(p1​)∩W(p2​))∪(R(p2​)∩W(p1​))∪(W(p1​)∩W(p2​))=∅$
则它们并发执行与时间无关，可以安全并发。

这个公式本质上检查三类冲突：
**读-写冲突**
$R(p1​)∩W(p2​)$
如果 `p1` 要读某个变量，而 `p2` 会写这个变量，那么谁先谁后会影响 `p1` 读到的值。

**写-读冲突**
$R(p2​)∩W(p1​)$
同理，如果 `p2` 要读某个变量，而 `p1` 会写这个变量，也会产生顺序依赖。

**写-写冲突**
$W(p1​)∩W(p2​)$
如果两个程序都写同一个变量，最后结果取决于谁后写。
